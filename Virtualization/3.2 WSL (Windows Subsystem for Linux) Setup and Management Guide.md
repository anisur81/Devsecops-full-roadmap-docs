
# Step by Step guide on WSL (Windows Subsystem for Linux) Setup and Management Guide

## Installation

Open PowerShell as Administrator and run:

```powershell
wsl --install
```

Check installed versions:

```powershell
wsl -l -v
```

## Install Specific Linux Distro (Optional)

### List available distros:

```powershell
wsl --list --online
```

### Install a specific distro:

```powershell
wsl --install -d Ubuntu-22.04
```

### Uninstall a distro:

```powershell
wsl --unregister Ubuntu-22.04
```

### Update WSL:

```powershell
wsl --update
```

## Proxy Configuration

### Check Windows WinHTTP proxy settings:

```powershell
netsh winhttp show proxy
```

**Example output:**
```
Current WinHTTP proxy settings:
    Direct access (no proxy server).
```

### Set proxy in WSL (temporary):

```bash
export http_proxy="http://10.20.0.60:8080"
export https_proxy="http://10.20.0.60:8080"
```

### Configure APT proxy:

```bash
sudo vi /etc/apt/apt.conf.d/95proxies
```

Add the following lines:

```
Acquire::http::Proxy "http://10.20.0.60:8080";
Acquire::https::Proxy "http://10.20.0.60:8080";
```

## Network Configuration

### Check network interfaces:

```bash
ifconfig -all
# or
ip a
```

## System Configuration

### Disable IPv6:

```bash
sudo cat /etc/sysctl.conf
```

Add these lines:

```
net.ipv6.conf.all.disable_ipv6=1
net.ipv6.conf.default.disable_ipv6=1
```

Apply changes:

```bash
sudo sysctl -p
```

### WSL Configuration (`/etc/wsl.conf`):

```bash
sudo cat /etc/wsl.conf
```

Example configuration:

```ini
[boot]
systemd=true

[user]
default=administrator

[network]
generateResolvConf=false
```

### DNS Configuration (`/etc/resolv.conf`):

```bash
sudo cat /etc/resolv.conf
```

Example configuration:

```
# nameserver 172.17.21.11
# nameserver 172.17.21.12
# search aibl.com.bd
8.8.8.8
1.1.1.1
```

### Make resolv.conf immutable:

```bash
sudo chmod 600 /etc/resolv.conf
sudo chattr +i /etc/resolv.conf
```

To remove the immutable attribute:

```bash
sudo chattr -i /etc/resolv.conf
```

### Hosts File (`/etc/hosts`):

```bash
administrator@WIN-RJ32TRRFOFP:~$ cat /etc/hosts
```

Example configuration:

```hosts
# This file was automatically generated by WSL. To stop automatic generation of this file, add the following entry to /etc/wsl.conf:
# [network]
# generateHosts = false
127.0.0.1       localhost
127.0.1.1       WIN-RJ32TRRFOFP.localdomain     WIN-RJ32TRRFOFP
10.20.0.60      dcproxy.aibl.com.bd
# The following lines are desirable for IPv6 capable hosts
::1     ip6-localhost ip6-loopback
fe00::0 ip6-localnet
ff00::0 ip6-mcastprefix
ff02::1 ip6-allnodes
ff02::2 ip6-allrouters
```

## Export and Import WSL Distros

### How to Export a WSL Distro

Exporting creates a snapshot of your entire Linux filesystem as a `.tar` file.

#### Step 1: List your distros

```powershell
wsl --list --verbose
```

#### Step 2: Shut down WSL

```powershell
wsl --shutdown
```

#### Step 3: Run the export command

```powershell
wsl --export <DistroName> <FilePath.tar>
```

**Example:**

```powershell
wsl --export Ubuntu-24.04 G:\WSL_IMAGE\ubuntu_2404_image_backup.tar
```

> **Tip:** Use `--format vhd` to export as a `.vhdx` file for faster backup/restore.

### How to Import a WSL Distro

Importing extracts your `.tar` file into a new distribution with a custom name and location.

#### Step 1: Create a folder for the distro

```powershell
mkdir D:\WSL\Ubuntu
```

#### Step 2: Run the import command

```powershell
wsl --import <NewDistroName> <InstallLocation> <PathToTarFile>
```

**Example:**

```powershell
wsl --import Ubuntu_24_inst2 G:\WSL_IMAGE\ubuntu_24_inst2 G:\WSL_IMAGE\ubuntu_2404_image_backup.tar
```

#### Step 3: Verify the import

```powershell
wsl --list --verbose
```

## Quick Reference Table

| Command | Description |
|---------|-------------|
| `wsl --install` | Install WSL with default Ubuntu distro |
| `wsl --install -d <DistroName>` | Install specific Linux distribution |
| `wsl -l -v` | List installed distros with versions |
| `wsl --list --online` | List available distros for installation |
| `wsl --unregister <DistroName>` | Uninstall/remove a distro |
| `wsl --update` | Update WSL components |
| `wsl --shutdown` | Shut down all running WSL instances |
| `wsl --export <Distro> <Path.tar>` | Export distro to tar file |
| `wsl --import <Name> <InstallPath> <TarFile>` | Import distro from tar file |

## Common WSL Configuration Files

| File | Purpose |
|------|---------|
| `/etc/wsl.conf` | WSL-specific settings (boot, user, network) |
| `/etc/resolv.conf` | DNS configuration |
| `/etc/hosts` | Static hostname mappings |
| `/etc/sysctl.conf` | Kernel parameter configuration |
| `/etc/apt/apt.conf.d/95proxies` | APT proxy settings |

## Troubleshooting Tips

1. **WSL not starting?** Run `wsl --shutdown` and restart
2. **DNS not working?** Check `/etc/resolv.conf` and ensure `generateResolvConf=false` in `wsl.conf`
3. **Systemd not running?** Ensure `systemd=true` is set in `[boot]` section of `/etc/wsl.conf`
4. **Proxy not working?** Verify proxy settings in both environment variables and APT configuration
