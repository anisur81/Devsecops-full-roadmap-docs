## Setup & Configuration

| Command | Description |
|---------|-------------|
| `git --version` | Show Git version |
| `git config --global user.name "Your Name"` | Set global username |
| `git config --global user.email "email@example.com"` | Set global email |
| `git config --global core.editor "code --wait"` | Set default editor |
| `git config --list` | Show all configuration settings |
| `git config --global --list` | Show global configuration |
| `git config --local --list` | Show local repository configuration |
| `git config --global alias.co checkout` | Create a command alias |
| `git config --global color.ui auto` | Enable colored output |

### Configuration Levels

| Level | Command Flag | Scope |
|-------|--------------|-------|
| System | `--system` | All users on the system |
| Global | `--global` | Current user (all repositories) |
| Local | `--local` | Current repository only |

## Creating & Cloning Repositories

| Command | Description |
|---------|-------------|
| `git init` | Initialize a new Git repository in current directory |
| `git init my-project` | Create a new directory and initialize Git |
| `git clone <url>` | Clone a remote repository |
| `git clone <url> my-folder` | Clone into a specific directory |
| `git clone --depth 1 <url>` | Shallow clone (only latest commit) |

## Basic Snapshotting

### Working with Files

| Command | Description |
|---------|-------------|
| `git status` | Show working directory status |
| `git status -s` | Short status format |
| `git add <file>` | Stage a specific file |
| `git add .` | Stage all changes (new, modified, deleted) |
| `git add -A` | Stage all changes (including untracked) |
| `git add -p` | Stage changes interactively (patch mode) |
| `git rm <file>` | Remove file from working directory and stage |
| `git rm --cached <file>` | Stop tracking file but keep it locally |
| `git mv <old> <new>` | Move/rename a file and stage the change |

### Committing

| Command | Description |
|---------|-------------|
| `git commit -m "message"` | Commit staged changes with message |
| `git commit -a -m "message"` | Stage all tracked files and commit |
| `git commit --amend` | Amend the last commit (add forgotten changes) |
| `git commit --amend -m "new message"` | Change last commit message |
| `git commit --amend --no-edit` | Add staged changes to last commit without editing message |
| `git commit -v` | Show diff in commit editor |

### Ignoring Files

Create `.gitignore` file in root directory:

```
# Ignore node_modules
node_modules/

# Ignore environment files
.env
.env.local

# Ignore build outputs
dist/
build/
*.exe
*.dll

# Ignore IDE files
.vscode/
.idea/
*.swp
```

| Command | Description |
|---------|-------------|
| `git check-ignore -v <file>` | Debug why a file is ignored |

## Branching & Merging

### Branch Management

| Command | Description |
|---------|-------------|
| `git branch` | List local branches (* marks current) |
| `git branch -r` | List remote branches |
| `git branch -a` | List all branches (local + remote) |
| `git branch <name>` | Create a new branch |
| `git branch -d <name>` | Delete a branch (safe) |
| `git branch -D <name>` | Force delete a branch |
| `git branch -m <old> <new>` | Rename a branch |
| `git branch -vv` | Show tracking branches |
| `git branch --merged` | Show branches merged into current |
| `git branch --no-merged` | Show branches not merged |
| `git branch --contains <commit>` | Show branches containing a commit |

### Switching Branches

| Command | Description |
|---------|-------------|
| `git switch <branch>` | Switch to an existing branch |
| `git switch -c <branch>` | Create and switch to new branch |
| `git switch -` | Switch to previous branch |
| `git checkout <branch>` | Traditional way to switch branches |
| `git checkout -b <branch>` | Create and switch (traditional) |

### Merging

| Command | Description |
|---------|-------------|
| `git merge <branch>` | Merge specified branch into current |
| `git merge --no-ff <branch>` | Merge with no fast-forward (preserve branch history) |
| `git merge --abort` | Abort a conflicting merge |
| `git merge --continue` | Continue merge after resolving conflicts |
| `git merge --squash <branch>` | Merge all commits as a single commit |
| `git merge <branch> --no-commit` | Merge without auto-committing |

### Merge Conflicts Resolution

When conflicts occur, Git marks files with:

```
<<<<<<< HEAD
Current branch changes
=======
Incoming branch changes
>>>>>>> branch-name
```

After resolving conflicts:

```bash
git add <resolved-file>
git commit -m "Merge resolution"
```

## Sharing & Updating

### Remote Operations

| Command | Description |
|---------|-------------|
| `git remote -v` | List remote repositories |
| `git remote add <name> <url>` | Add a new remote |
| `git remote remove <name>` | Remove a remote |
| `git remote rename <old> <new>` | Rename a remote |

| `git remote prune <name>` | Remove stale remote-tracking branches |

### Fetching, Pulling, Pushing

| Command | Description |
|---------|-------------|

| `git fetch --all` | Fetch from all remotes |
| `git fetch <remote> <branch>` | Fetch a specific branch |

### Viewing History

| Command | Description |
|---------|-------------|

| `git show <commit>:<file>` | Show file as it was at a commit |

## Undoing Changes

### Working Directory (Unstaged)

| Command | Description |
|---------|-------------|
| `git restore <file>` | Discard changes in working directory |
| `git restore .` | Discard all local changes |
| `git checkout -- <file>` | Traditional way to discard changes |
| `git clean -n` | Dry run - show what would be removed |
| `git clean -f` | Remove untracked files |
| `git clean -fd` | Remove untracked files and directories |
| `git clean -fdx` | Remove ignored files too |

### Staging Area (Staged but not committed)

| Command | Description |
|---------|-------------|
| `git restore --staged <file>` | Unstage a file (keep changes) |
| `git reset HEAD <file>` | Traditional way to unstage |
| `git reset --soft HEAD~1` | Undo last commit, keep changes staged |
| `git reset --mixed HEAD~1` | Undo last commit, unstage changes |
| `git reset --hard HEAD~1` | Undo last commit, discard changes (dangerous) |

### Resetting

| Command | Description |
|---------|-------------|
| `git reset --soft <commit>` | Move HEAD, keep changes staged |
| `git reset --mixed <commit>` | Move HEAD, keep changes unstaged |
| `git reset --hard <commit>` | Move HEAD, discard all changes |
| `git reset --hard HEAD` | Discard all staged and unstaged changes |
| `git revert <commit>` | Create new commit that undoes a previous commit |
| `git revert HEAD~1..HEAD` | Revert a range of commits |

### The Difference: reset vs revert

| `git reset` | `git revert` |
|-------------|---------------|
| Rewrites history | Creates new commit |
| Dangerous for shared branches | Safe for shared branches |
| Use for local commits only | Use for public commits |

## Stashing

| Command | Description |
|---------|-------------|
| `git stash` | Save uncommitted changes (staged and unstaged) |
| `git stash push -m "message"` | Stash with a descriptive message |
| `git stash list` | List all stashes |
| `git stash show` | Show latest stash summary |
| `git stash show -p` | Show full diff of latest stash |
| `git stash pop` | Apply latest stash and remove from stash list |
| `git stash apply` | Apply latest stash (keep in stash list) |
| `git stash apply stash@{2}` | Apply a specific stash |
| `git stash drop` | Remove latest stash |

| `git stash branch <branch>` | Create branch from stash |
| `git stash --keep-index` | Stash untracked only (keep staged changes) |
| `git stash -u` | Stash untracked files as well |

## Rebasing

### Basic Rebase

| Command | Description |
|---------|-------------|
| `git rebase <branch>` | Rebase current branch onto another branch |
| `git rebase <branch> --onto <newbase>` | Rebase onto different base |
| `git rebase --continue` | Continue rebase after resolving conflicts |
| `git rebase --skip` | Skip current commit during rebase |
| `git rebase --abort` | Abort rebase and restore previous state |
| `git rebase -i HEAD~n` | Interactive rebase for last n commits |

### Interactive Rebase Commands

When running `git rebase -i`, you can use these commands:

| Command | Abbreviation | Description |
|---------|--------------|-------------|
| `pick` | `p` | Use commit (default) |
| `reword` | `r` | Change commit message |
| `edit` | `e` | Stop to amend commit content |
| `squash` | `s` | Combine with previous commit |
| `fixup` | `f` | Combine, discarding message |
| `drop` | `d` | Remove commit |
| `reword` | `r` | Change commit message |
| `exec` | `x` | Run shell command |

### Example Interactive Rebase

```bash
git rebase -i HEAD~3
```

Editor shows:
```
pick f7f3f6d Add feature A
pick 310154e Fix bug
pick a5f4a0d Add feature B
```

Change to:
```
reword f7f3f6d Add feature A
fixup 310154e Fix bug
pick a5f4a0d Add feature B
```

### Rebase vs Merge

| `git merge` | `git rebase` |
|-------------|---------------|
| Preserves complete history | Creates linear history |
| Adds merge commits | No extra merge commits |
| Safer for shared branches | Cleaner, but rewrites history |
| Better for feature branches | Better for updating branches |

## Tags

| Command | Description |
|---------|-------------|
| `git tag` | List all tags |
| `git tag -l "v1.*"` | List tags matching pattern |
| `git tag <tagname>` | Create lightweight tag at current commit |
| `git tag -a <tagname> -m "message"` | Create annotated tag |
| `git tag -a <tagname> <commit>` | Tag a specific commit |
| `git show <tagname>` | Show tag details |
| `git tag -d <tagname>` | Delete local tag |
| `git push origin <tagname>` | Push a specific tag |
| `git push --tags` | Push all tags |
| `git push origin --delete <tagname>` | Delete remote tag |
| `git checkout <tagname>` | Check out a tag (detached HEAD) |

### Lightweight vs Annotated Tags

| Aspect | Lightweight | Annotated |
|--------|-------------|-----------|
| Storage | Just a pointer | Full Git object |
| Metadata | No | Contains tagger, date, message |
| PGP Signing | No | Yes |
| Recommendation | Private/temporary releases | Public releases |

## Advanced Commands

### Bisect (Find Bugs)

| Command | Description |
|---------|-------------|
| `git bisect start` | Start bisect session |
| `git bisect bad` | Mark current commit as bad |
| `git bisect good <commit>` | Mark commit as good (reference) |
| `git bisect reset` | End bisect session |
| `git bisect log` | Show bisect log |
| `git bisect visualize` | Visualize remaining commits |
| `git bisect run <script>` | Automate bisect with test script |

### Reflog (Recover Lost Commits)

| Command | Description |
|---------|-------------|
| `git reflog` | Show reference log (where HEAD has been) |
| `git reflog show <branch>` | Show reflog for specific branch |
| `git reflog expire --expire=now --all` | Clear reflog |
| `git checkout HEAD@{2}` | Check out a previous HEAD position |
| `git reset --hard HEAD@{5}` | Reset to a previous HEAD position |

### Hooks

Git hooks are scripts in `.git/hooks/`:

| Hook Name | When It Runs |
|-----------|--------------|
| `pre-commit` | Before commit (useful for linting) |
| `prepare-commit-msg` | Before commit message editor |
| `commit-msg` | After commit message created |
| `post-commit` | After commit |
| `pre-push` | Before git push |
| `post-checkout` | After git checkout |
| `post-merge` | After git merge |
| `pre-rebase` | Before git rebase |

### Filter Branch (Rewriting History)

| Command | Description |
|---------|-------------|
| `git filter-branch --tree-filter <command>` | Modify file tree |
| `git filter-branch --env-filter <command>` | Modify environment |
| `git filter-branch --commit-filter <command>` | Modify commits |
| `git filter-branch --tag-name-filter <command>` | Modify tags |

> **Warning:** Use `git filter-repo` (modern alternative) instead of `filter-branch`.

## Aliases

### Creating Aliases

```bash
# Basic aliases
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
git config --global alias.unstage 'reset HEAD --'

# Advanced aliases
git config --global alias.last 'log -1 HEAD'
git config --global alias.tree 'log --graph --oneline --decorate --all'
git config --global alias.uncommit 'reset --soft HEAD~1'
git config --global alias.cleanup '!git branch --merged | grep -v "\*\|main\|master" | xargs -n 1 git branch -d'
```

### Common Alias Examples

```bash
# Show pretty log tree
git config --global alias.tree "log --graph --pretty=format:'%C(yellow)%h%C(cyan)%d%Creset %s %C(white)- %an, %ar%Creset' --all"

# Show changed files in last commit
git config --global alias.changed "show --name-only"

# Show all aliases
git config --global alias.aliases "config --get-regexp alias"

# Undo last commit (keep changes)
git config --global alias.undo "reset --soft HEAD~1"

# Amend with staged changes
git config --global alias.amend "commit --amend --no-edit"
```

## Error Messages & Solutions

| Error | Solution |
|-------|----------|
| `fatal: not a git repository` | Run `git init` or navigate to correct repo |
| `cannot lock ref` | Delete conflicting ref or run `git gc` |
| `merge conflict` | Manually resolve conflicts, then `git add` |
| `detached HEAD` | Create branch: `git switch -c <branch-name>` |
| `failed to push some refs` | Run `git pull` first to integrate changes |
| `fatal: refusing to merge unrelated histories` | Use `git pull --allow-unrelated-histories` |
| `Your branch is ahead of 'origin/main'` | Run `git push` |
| `permission denied (publickey)` | Add SSH key to GitHub/GitLab |
