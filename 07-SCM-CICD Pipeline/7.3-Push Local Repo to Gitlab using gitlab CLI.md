## 1. Install GitLab CLI (glab) on Ubuntu

### Download the GitLab CLI Package

```bash
wget https://gitlab.com/gitlab-org/cli/-/releases/v1.94.0/downloads/glab_1.94.0_linux_amd64.deb
```

### Install the Package

```bash
sudo dpkg -i glab_1.94.0_linux_amd64.deb
```

### Fix Dependency Issues (If Needed)

```bash
sudo apt --fix-broken install -y
```

### Verify Installation

```bash
glab --version
```

Expected output:
```
glab version 1.94.0
```

### Install xdg-utils (Recommended)

```bash
sudo apt install xdg-utils -y
```

## 2. Create a New Project on GitLab

1. Log in to [GitLab.com](https://gitlab.com)
2. Click **New project** (or the **+** icon → **New project**)
3. Choose **Create blank project** (most common)
4. Fill in the project details:

| Field | Example Value |
|-------|---------------|
| Project name | `django-api-project` |
| Project slug | Auto-generated |
| Visibility Level | `Private` (recommended) or `Public` |

5. **Do NOT initialize with README** (leave blank)
6. Click **Create project**

> **Note:** Create the project as a blank project without `README.md` or `.gitlab-ci.yml` file.

## 3. Generate Personal Access Token (PAT)

### Navigate to Token Creation Page

Go to: `https://gitlab.com/-/user_settings/personal_access_tokens`

Or:
1. Click your **avatar** (top-right corner)
2. Go to **Preferences** → **Access Tokens**
3. Click **Add new token**

### Fill Token Details

| Field | Value |
|-------|-------|
| **Token name** | `my-dev-token` (or any descriptive name) |
| **Expiration date** | Select a date (e.g., 1 year from now) |
| **Description (optional)** | `Token for pushing code from local machine` |

### Select Scopes

Select the following scopes for full repository access:

| Scope | Purpose |
|-------|---------|
| `read_user` | Read user information |
| `read_repository` | Read repository contents |
| `write_repository` | Write to repository (push code) |
| `read_registry` | Read container registry |
| `write_registry` | Write to container registry |
| `api` | Full API access |
| `read_api` | Read-only API access |
| `create_runner` | Create CI runners |
| `manage_runner` | Manage CI runners |
| `k8s_proxy` | Kubernetes proxy access |
| `ai_features` | AI features access |
| `self_rotate` | Self-rotate token |

> **Minimum required scopes:** `api` and `write_repository`

### Generate and Copy Token

Click **Create personal access token**

**⚠️ Important:** Copy the token immediately. You won't be able to see it again!

Example token format:
```
glpat-fOR13vPGcEZUYceUOvK-8mM6MQpvOjEKdTo4cGs2NA8.01.170qg97is
```

## 4. Login to GitLab with glab

Run the login command:

```bash
glab auth login
```

### Step-by-Step Process

```
- Signing into gitlab.com

The minimum required scopes are 'api' and 'write_repository'.
Generate a personal access token at https://gitlab.com/-/user_settings/personal_access_tokens?scopes=api,write_repository
- glab config set -h gitlab.com git_protocol https
✓ Configured Git protocol.
- glab config set -h gitlab.com api_protocol https
✓ Configured API protocol.
✓ Logged in as anisur81
✓ Configuration saved to /home/administrator/.config/glab-cli/config.yml
  - Host: gitlab.com
```

You will be prompted to enter your Personal Access Token. Paste the token you generated.

### Verify Login

```bash
glab auth status
```

Expected output:
```
gitlab.com
  ✓ Logged in to gitlab.com as anisur81
  ✓ Git operations protocol: https
  ✓ API protocol: https
```

## 12. Troubleshooting

### Remote Already Exists

**Check current remote:**
```bash
git remote -v
```

**Remove existing origin:**
```bash
git remote remove origin
```

**Add correct origin:**
```bash
git remote add origin https://gitlab.com/yourusername/yourproject.git
```

### Authentication Failed

**Solution 1:** Re-login with glab
```bash
glab auth logout
glab auth login
```

**Solution 2:** Generate a new PAT with correct scopes
- Ensure `api` and `write_repository` scopes are selected

### Push Rejected (Non-fast-forward)

```bash
git pull origin main --rebase
git push origin main
```

### SSL Certificate Error (Corporate Networks)

```bash
git config --global http.sslVerify false
```

> **Warning:** Only use this in trusted environments.

### Check Git Configuration

```bash
git config --list
```

### Verify Remote URL is Correct

```bash
git remote -v
```

Should show your GitLab repository URL.

## 13. Docker Commands Reference

Here are useful Docker commands for managing container images from GitLab Container Registry:

### Image Management

| Command | Description |
|---------|-------------|
| `sudo docker images` | List all Docker images |
| `sudo docker rmi -f 5877daa101c6` | Force remove image by ID |
| `sudo docker pull registry.gitlab.com/anisur81/portfolioci:latest` | Pull image from GitLab Container Registry |

### Container Management

| Command | Description |
|---------|-------------|
| `sudo docker ps` | List running containers |
| `sudo docker ps -a` | List all containers (including stopped) |
| `sudo docker container prune -f` | Remove all stopped containers |
| `sudo docker start 98dad68118cd` | Start a stopped container by ID |
| `sudo docker stop <container-id>` | Stop a running container |

### Running Containers from GitLab Registry

```bash
# Pull the latest image
sudo docker pull registry.gitlab.com/anisur81/portfolioci:latest

# Run container on port 8005
sudo docker run -d -p 8005:8000 registry.gitlab.com/anisur81/portfolioci:latest
```

### Complete Docker Workflow Example

```bash
# Pull the image
sudo docker pull registry.gitlab.com/anisur81/portfolioci:latest

# Check if container exists and is stopped
sudo docker ps -a

# Start existing container
sudo docker start 98dad68118cd

# Or run a new container
sudo docker run -d -p 8005:8000 registry.gitlab.com/anisur81/portfolioci:latest

# Verify container is running
sudo docker ps
```

## Environment Variables (Optional)

For automation, you can set the PAT as an environment variable:

```bash
export GITLAB_TOKEN="glpat-your-token-here"
```

Then use it in scripts:

```bash
glab auth login --stdin < $GITLAB_TOKEN
```
