## Step 1: Create Empty Repository on GitHub

1. Go to [GitHub.com](https://github.com) and log in
2. Click the **+** icon in the top-right corner → **New repository**
3. Fill in the details:

| Field | Example Value |
|-------|---------------|
| Repository name | `my-devsecops-portfolio` |
| Visibility | Public or Private |
| Initialize with README | **❌ Do NOT check** |

4. Click **Create repository**

> **Note:** Do NOT initialize with README, .gitignore, or license. We'll push an existing repository.

## Step 4: Login to GitHub with gh

### Run Login Command

```bash
gh auth login
```

### Step-by-Step Answers

| Prompt | Your Answer |
|--------|-------------|
| What account do you want to log into? | `GitHub.com` |
| What is your preferred protocol for Git operations? | `HTTPS` |
| How would you like to authenticate GitHub CLI? | `Login with a web browser` |

### Complete the Browser Authentication

1. A code will appear in the terminal, like: `8FGH-1234`
2. Open: `https://github.com/login/device`
3. Paste the code
4. Click **Authorize GitHub CLI**

### Confirm Login

```bash
gh auth status
```

Expected output:
```
github.com
  ✓ Logged in to github.com account anisur81 (/home/administrator/.config/gh/hosts.yml)
  - Active account: true
  - Git operations protocol: https
  - Token: gho_************************************
  - Token scopes: 'gist', 'read:org', 'repo', 'workflow'
```

### Configure Git to Use GitHub CLI Automatically

```bash
gh auth setup-git
```

This configures Git so that:
- ✅ No manual token entry required
- ✅ No password prompts
- ✅ Seamless push/pull operations

## Step 6: Configure Git (First Time Only)

Check current configuration:

```bash

```

Set your username and email (use your GitHub credentials):

```bash
git config --global user.name "anisur81"
git config --global user.email "anisur81@gmail.com"
```

Set VS Code as the default Git editor:

```bash
git config --global core.editor "code --wait"
```

### Verify Configuration

```bash
git config --list
```

Expected output:
```

user.email=anisur81@gmail.com
core.editor=code --wait
credential.https://github.com.helper=!/usr/bin/gh auth git-credential
...
```

## Step 12: Troubleshooting

### Remote Already Exists

**Check current remote:**
```bash
git remote -v
```

**Remove existing origin:**
```bash
git remote remove origin
```

**Add correct origin:**
```bash
git remote add origin https://github.com/yourusername/my-devsecops-portfolio.git
```

### Change Remote URL (set-url)

If you already have a remote named `origin` but need to change the URL:

```bash
git remote set-url origin https://github.com/anisur81/portfolio.git
```

Use `set-url` when you want to:
- Switch from SSH → HTTPS
- Change from old repo → new repo
- Correct a wrong URL

### Clear Cached Credentials (Windows/WSL Common)

If you're using Windows + WSL (paths like `/mnt/g/...`), old credentials may be cached:

```bash
git config --global --unset credential.helper
```

### Verify Remote After Fix

```bash
git remote -v
```

Expected:
```
origin  https://github.com/anisur81/devsecops-portfolio.git (fetch)
origin  https://github.com/anisur81/devsecops-portfolio.git (push)
```

## Alternative: SSH Authentication

If you prefer SSH over HTTPS, follow these steps.

### Step 1: Check Existing SSH Keys

```bash
ls -al ~/.ssh
```

If you see files like `id_ed25519` and `id_ed25519.pub`, you may already have keys.

### Step 2: Generate New SSH Key

Use your GitHub email:

```bash
ssh-keygen -t ed25519 -C "anisur81@gmail.com"
```

Press Enter to accept the default file location and passphrase (or set a passphrase for extra security).

### Step 3: Start SSH Agent

```bash
eval "$(ssh-agent -s)"
```

### Step 4: Add Key to Agent

```bash
ssh-add ~/.ssh/id_ed25519
```

### Step 5: Copy Public Key

```bash
cat ~/.ssh/id_ed25519.pub
```

Copy the full output starting with `ssh-ed25519 AAAA...`

### Step 6: Add Key to GitHub

1. Go to **GitHub** → **Settings** → **SSH and GPG keys**
2. Click **New SSH key**
3. Paste your copied key
4. Click **Add SSH key**

### Step 7: Test Connection

```bash
ssh -T git@github.com
```

**First time prompt:**
```
Are you sure you want to continue connecting (yes/no)?
```

Type `yes`

**Expected success output:**
```
Hi anisur81! You've successfully authenticated...
```

### Step 8: Change Git Remote to SSH

Inside your project:

```bash
git remote set-url origin git@github.com:anisur81/devsecops-portfolio.git
```

### Step 9: Verify Remote

```bash
git remote -v
```

Expected output:
```
origin  git@github.com:anisur81/devsecops-portfolio.git (fetch)
origin  git@github.com:anisur81/devsecops-portfolio.git (push)
```

### Step 10: Push Code

```bash
git push -u origin main
```

🎉 **Done! You're now using SSH authentication.**

## Docker Commands Reference

Here are useful Docker commands for managing container images (from your examples):

### Image Management

| Command | Description |
|---------|-------------|
| `sudo docker images` | List all Docker images |
| `sudo docker rmi -f 5877daa101c6` | Force remove image by ID |
| `docker pull ghcr.io/anisur81/portfolio:tag` | Pull image from GitHub Container Registry |
| `docker pull registry.gitlab.com/anisur81/portfolioci:latest` | Pull image from GitLab Registry |

### Container Management

| Command | Description |
|---------|-------------|
| `sudo docker ps` | List running containers |
| `sudo docker ps -a` | List all containers (including stopped) |
| `sudo docker container prune -f` | Remove all stopped containers |
| `sudo docker run -it --entrypoint sh image:tag` | Run container interactively with shell |
| `sudo docker run -d -p 8002:8000 image:tag` | Run container in detached mode with port mapping |

### Tagging Images

```bash
sudo docker tag source-image:old-tag target-image:new-tag
```

Example:
```bash
sudo docker tag ghcr.io/anisur81/djangoportfolio:3ce1cb378e0988b06d7e9c2712899d3c659a36eb ghcr.io/anisur81/djangoportfolio:latest
```

### Running Containers

```bash
# Run Django portfolio container on port 8002
sudo docker run -d -p 8002:8000 ghcr.io/anisur81/djangoportfolio:latest

# Run GitLab portfolio container on port 8003
sudo docker run -d -p 8003:8000 registry.gitlab.com/anisur81/portfolioci:latest
```
