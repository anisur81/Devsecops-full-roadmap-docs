# 2.3.2  Brief Discussion for Linux ifconfig Command with Examples

---

## Introduction

The `ifconfig` command is a part of **net-tools**, a legacy Linux tool for configuring a network interface. Modern distributions use the `ip` command, which works in a similar manner.

Even though `ifconfig` has limited capabilities compared to `ip`, the command is still commonly used to configure a network interface in Linux.

This tutorial will teach you how to use the Linux `ifconfig` command through practical examples.

---

## Prerequisites

- A Linux system
- Access to the terminal
- `sudo` privileges when making updates to network interface settings

> **Important:** `ifconfig` used to come preinstalled on all major Linux distributions. However, the tool was deprecated due to lack of maintenance and support for IPv6. The `ip` command is the preferred alternative.

---

## Linux ifconfig Command Syntax

The basic `ifconfig` syntax looks like this:

```bash
ifconfig [interface_name_optional] [arguments_optional]
```

The `ifconfig` command prints an output when executed without arguments or with the interface name only. If used with arguments, there is no output.

---

## Linux ifconfig Command Options

Below is a list of common `ifconfig` command options:

| Option | Description |
|--------|-------------|
| `-s` | Displays only a shortlist of all interfaces |
| `-v` | Prints a more detailed configuration for all interfaces |
| `interface_name` | Represents the interface's name and is accompanied by a unit number. Prints the configuration for a specific interface |
| `down` | Deactivates the interface |
| `up` | Activates the interface |
| `[IP_address_number]` | Assigns an IP address to a specified interface |
| `netmask [address]` | Changes the network mask for a specific interface |
| `broadcast [address]` | Sets the broadcast address for a specific interface |
| `mtu [number]` | Sets the Maximum Transfer Unit (MTU) of an interface |
| `promisc` | Enables promiscuous mode in which the interface receives all packets |
| `[-]promisc` | Disables promiscuous mode |
| `arp` | Enables the ARP protocol on a specific interface |
| `[-]arp` | Disables the ARP protocol on a specific interface |
| `allmulti` | Enables the all-multicast mode in which the interface receives all multicast packets |
| `[-]allmulti` | Disables the all-multicast mode |

## 19 Practical Examples

### 1. View Active Network Interface Settings

Run `ifconfig` with no arguments to print essential information about all active network interfaces:

```bash
ifconfig
```

**Sample output:**
```
enpo3: flags=4163<UP,BROADCAST,RUNNING,MULTICAST> mtu 2000
        inet 10.0.2.15 netmask 255.0.0.0 broadcast 10.255.255.255
        inet6 fe80::c912:d4bf:3acb:f531 prefixlen 64 scopoid 0x20<link>
        ether 08:00:27:d6:51:a6 txqueuelen 1000 (Ethernet)
        RX packets 1442 bytes 632447 (632.4 KB)
        RX errors 0 dropped 0 overruns 0 frame 0
        TX packets 1540 bytes 149029 (149.0 KB)
        TX errors 0 dropped 0 overruns 0 carrier 0 collisions 0

lo: flags=73<UP,LOOPBACK,RUNNING> mtu 65536
        inet 127.0.0.1 netmask 255.0.0.0
        inet6 ::1 prefixlen 128 scopoid 0x10<host>
        loop txqueuelen 1000 (Local Loopback)
        RX packets 2012 bytes 180484 (180.4 KB)
        RX errors 0 dropped 0 overruns 0 frame 0
        TX packets 2012 bytes 180484 (180.4 KB)
        TX errors 0 dropped 0 overruns 0 carrier 0 collisions 0
```

The output shows crucial data about the network interfaces, like:
- The names of the active network interfaces (e.g., `enpo3` and `lo`)
- The hardware MAC address
- The IP address (`inet`), netmask, and broadcast address
- The MTU (Maximum Transmission Unit) value

---

### 2. Display a Shortlist of Active Interfaces

Use the `-s` flag with `ifconfig` to display a concise summary of every active interface:

```bash
ifconfig -s
```

**Sample output:**
```
Iface      MTU RX-OK RX-ERR RX-DRP RX-OVR TX-OK TX-ERR TX-DRP TX-OVR
enp0s3    2000   879      0      0      0   941      0      0      0
lo       65536   464      0      0      0   464      0      0      0
```

---

### 3. Print the Verbose Output

Use the verbose option `-v` to print a more detailed output:

```bash
ifconfig -v
```

Depending on the system, the outcome is either the same as `ifconfig` without arguments or slightly more in-depth.

---

### 4. Show Every Network Interface Configuration

To get info about both active and inactive network interfaces, run `ifconfig` with the `-a` option:

```bash
ifconfig -a
```

---

### 5. Get Information About a Specific Network Interface

To get information about a single specific network interface:

```bash
ifconfig [interface_name]
```

**Example:**
```bash
ifconfig enp0s3
```

---

### 6. Disable a Network Interface

To deactivate a specific network interface, use the `down` flag with `sudo`:

```bash
sudo ifconfig enp0s3 down
```

Run `ifconfig` without any arguments to verify the interface is no longer active.

---

### 7. Enable a Network Interface

To reactivate a network interface, use the `up` flag with `sudo`:

```bash
sudo ifconfig enp0s3 up
```

Verify with:
```bash
ifconfig
```

---

### 8. Filter IP Addresses Using grep

To list only IP addresses, use `grep`:

```bash
ifconfig | grep inet
```

---

### 9. Assign an IP Address to an Interface

To define an IP address for a specific interface:

```bash
sudo ifconfig [interface_name] [IP_address]
```

**Example:**
```bash
sudo ifconfig enp0s3 10.0.2.20
```

Run `ifconfig` with no arguments to confirm the IP address change.

---

### 10. View Network Interface Masks

Each interface has its assigned netmask. View them with:

```bash
ifconfig
```

**Netmask classes:**
- Class A: `255.0.0.0`
- Class B: `255.255.0.0`
- Class C: `255.255.255.0`

---

### 11. Assign a Netmask to an Interface

```bash
sudo ifconfig [interface_name] netmask [netmask_IP]
```

**Example:**
```bash
sudo ifconfig enp0s3 netmask 255.255.255.0
```

---

### 12. View the Broadcast Address of Interfaces

Use `grep` to list the broadcast addresses:

```bash
ifconfig | grep broadcast
```

---

### 13. Assign a New Broadcast IP to a Network Interface

```bash
sudo ifconfig [interface_name] broadcast [broadcast_IP]
```

**Example:**
```bash
sudo ifconfig enp0s3 broadcast 10.0.2.250
```

---

### 14. Configure Multiple Network Parameters with One Command

The IP address, netmask, and broadcast can all be modified with one command:

```bash
sudo ifconfig [interface_name] [IP] netmask [netmask_address] broadcast [broadcast_address]
```

**Example:**
```bash
sudo ifconfig enp0s3 10.0.2.15 netmask 255.0.0.0 broadcast 10.0.2.255
```

---

### 15. Set MTU for an Interface

```bash
sudo ifconfig [interface_name] mtu [mtu_value]
```

**Example:**
```bash
sudo ifconfig enp0s3 mtu 2000
```

> **Note:** Not all network interfaces allow the MTU to be changed.

---

### 16. Set an Alias IP

`ifconfig` allows you to associate additional IP addresses with an interface using an IP alias. The alias IP must belong to the same netmask.

**Example - create an alias:**
```bash
sudo ifconfig enp0s3:0 10.0.2.30
```

**Remove an alias:**
```bash
sudo ifconfig enp0s3:0 down
```

---

### 17. Enable Promiscuous Mode

Promiscuous mode allows a network device to accept all packets (often used to analyze network activity):

```bash
sudo ifconfig enpo3 promisc
```

Verify with `ifconfig` - the `PROMISC` flag should appear.

---

### 18. Disable Promiscuous Mode

```bash
sudo ifconfig enpo3 -promisc
```

---

### 19. View Transmission Errors

Print any transmission errors by filtering the output with `grep`:

```bash
ifconfig | grep errors
```
