## What is an Operating System?

The operating system creates and runs the programs that make up your computer applications, serving as the backbone of your computer's functionality.

In other words, the operating system acts as an intermediary between the hardware and the applications that run on the computer. It is responsible for managing and coordinating the use of the computer's hardware resources, such as memory, processing power, and input/output devices, as well as providing a common interface for applications to interact with the hardware.

Through this, applications can access the necessary resources they need to function properly. The OS makes sure everything is running smoothly and coordinates all the different parts of the computer.

---

## History of Operating Systems

It all begins a long way back from the early days of batch processing on centralized mainframes, which have evolved through time. This evolution started with the introduction of time-sharing systems, allowing multiple users to access the computer at once.

- **MS-DOS** was the first popular personal computer operating system.
- **Macintosh** (introduced in 1984) revolutionized the game with the introduction of the graphical user interface (GUI).
- In the 90s, **Windows**, introduced by Microsoft, combined features of the Macintosh with MS-DOS and became the most popular personal computer operating system.

## Functions of an Operating System

### Memory Assignment
To guarantee an appropriate amount of memory for each program in operation and to resolve issues between programs, the system takes care of the computer's memory management and assignment through proper handling.

### Resource Allocation
Responsible for coordinating and efficiently utilizing various computer resources, such as CPU time, memory, and storage. Allocates resources with the goal of achieving optimal performance.

It is the central authority that allows each program to run efficiently and without interfering with other programs and processes.

### Safety and Protection
To keep the computer and its information secure, the operating system comes with safety features like user authentication, access control, and data protection.

### File Organization
File system management takes care of organizing and managing how the computer stores, retrieves, and changes files and information.

---

## Types of Operating Systems

Operating systems come in various shapes and sizes, each with its own unique set of features and abilities. Broadly speaking, the main categories are:

- **Single-tasking operating systems**: Found on older computers, capable of running only one program at a time. More user-friendly and less prone to crashes.
- **Multitasking operating systems (Real-time systems)**: Quickly react to external events. Often used in aviation, driving, and factory automation.

---

## Process Management

Process management is a crucial aspect of operating systems and regards the way programs and processes are managed on a computer. The goal is to use system resources efficiently.

A process management system takes care of managing the life cycle of each process, including:
- Allocating resources
- Scheduling
- Synchronization
- Termination of processes

---

## Memory Management

Memory management deals with the management and coordination of computer memory. This is done so that each process has enough memory to execute its tasks and the system remains stable.

Memory management involves:
- Allocating memory to processes
- Freeing up memory that is no longer needed
- Ensuring no conflicts between processes fighting for memory resources

The goal is to make the most effective use of memory resources, provide fast memory access, and minimize memory waste and fragmentation.

---

## Storage Management

Storage management refers to the process of monitoring and controlling the storage of data in computer systems, with the goal of making the best use of available storage resources and minimizing the risk of data loss or corruption.

This includes tasks such as allocating, organizing, and monitoring the use of storage resources.

---

## File System Management

The file system determines the structure and organization of files and directories, as well as how data is stored and retrieved. Common file systems include **NTFS** and **ext4**.

The main goal of File System Management is to provide a structured and organized way of storing and accessing data on a computer. This includes:
- Creating and deleting files
- Creating and managing directories
- Managing the allocation of storage space

---

## Security and Access Control

Security Access Control (SAC) is the process of protecting and controlling access to resources and data stored on a computer system. This includes:
- Managing user accounts
- Setting permissions for access to files and directories
- Controlling the execution of programs

---

## Networking

Networking refers to the process of connecting computers and devices to each other to exchange data and information (Client-Server Architecture). This includes:
- Configuring network interfaces
- Setting up communication protocols
- Managing the flow of data between devices

Networking plays a key role in enabling communication and collaboration between different devices and systems.

---

## Interrupts and Exception Handling

These are methods used by the computer system to handle unexpected events or errors.

- **Interrupts**: Signals sent to the processor indicating that an event requiring immediate attention has occurred (e.g., key press, mouse click).
- **Exception handling**: The process of dealing with errors or unexpected events during program execution (e.g., divide-by-zero, illegal memory accesses).

---

## Deadlocks and Resource Allocation

Deadlocks occur because multiple processes need access to the same resources at the same time. If two processes each hold onto a resource that the other process needs, they can get stuck waiting for each other, causing a deadlock.

This usually happens when two or more processes are blocked, and each is waiting for the release of a resource held by the other process. This creates a **circular wait condition** where processes are unable to proceed.

---

## Processor Scheduling

Process Scheduling refers to determining which tasks the processor will execute next and when. This involves making decisions about how to allocate the processor's time among waiting tasks based on factors such as priority, deadline, and other resource requirements.

The operating system uses algorithms and data structures to manage the scheduling process efficiently.

---

## Process Synchronization

Process synchronization avoids conflicts and race conditions by ensuring that access to shared resources (data or files) is regulated so that only one process at a time can access them.

This is achieved through synchronization techniques such as:
- Semaphores
- Monitors
- Critical sections

---

## Threads

The concept of threads in an operating system is about making sure multiple tasks can run in parallel within a single process. This allows for more efficient use of the processor's time and can lead to improved performance.

> **Note:** A process is just a program that is executing, and it can contain multiple threads. Each thread runs independently and has its own program counter, stack, and local variables.

---

## Virtual Memory

The purpose of virtual memory is to allow a computer system to run multiple programs simultaneously while ensuring each program has enough memory to execute properly.

Each program has its own **virtual address space**, isolated from the memory of other programs. The operating system allocates space on the hard disk to be used as an extension of RAM, acting as temporary storage for data that is not currently needed.

---

## File System Implementation

File system implementation involves organizing storage of files and directories efficiently and managing allocation of storage space. This includes:
- Creating and deleting files
- Creating and managing directories
- Using index allocation techniques

The implementation must ensure data remains secure with proper access control and protection mechanisms.

---

## OS-Level Services

System services are facilities and functions provided by the operating system to users and applications to make it easier to interact with hardware. These services are provided at the abstraction level, hiding the complexities of the underlying hardware.

Services include:
- Memory management
- Process management
- File management

These services are designed to be flexible and customizable.

---

## Design Principles

Operating System design principles are guidelines that designers follow to make the OS efficient, reliable, user-friendly, and secure.

### Modularity
Divide the system into smaller, manageable components that can be developed, tested, and maintained independently.

### Abstraction
Hide the complexity of the underlying hardware and present a simple, unified view to the user.

### Hierarchy
Organize the system into a series of levels, each building upon the services of the lower level.

---

## Kernel Interfaces and System Utilities

System utilities are programs used for various tasks such as file management, process management, and system maintenance. They run outside of the kernel and interact with it through kernel interfaces.

The kernel interfaces provide a standard way for system utilities to access kernel services.
