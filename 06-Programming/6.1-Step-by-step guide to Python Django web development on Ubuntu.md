 # Django Installation and First Project Setup on Ubuntu

This guide walks you through installing Django, creating a virtual environment, setting up your first project, and creating a basic app with views and templates.

---

## Prerequisites

- Ubuntu operating system (20.04, 22.04, or 24.04)
- Internet connection
- Basic familiarity with terminal commands

---

## Step 1: Install Required Packages

Open terminal and run the following commands to install Python and pip:

```bash
sudo apt update
sudo apt install python3 python3-pip python3-venv -y
```

### Verify Installation

Check that Python and pip were installed correctly:

```bash
python3 --version
```

Expected output:
```
Python 3.10.12
```

```bash
pip3 --version
```

Expected output:
```
pip 22.0.2 from /usr/lib/python3/dist-packages/pip (python 3.10)
```

---

## Step 2: Create a Virtual Environment

A virtual environment keeps your project dependencies isolated from the system Python.

### Create Project Directory

```bash
mkdir myproject
cd myproject
```

### Create and Activate Virtual Environment

```bash
python3 -m venv venv
source venv/bin/activate
```

After activation, you'll see `(venv)` at the beginning of your terminal prompt:

```
(venv) user@host:~/myproject$
```

### Deactivate (When Needed)

To exit the virtual environment later:

```bash
deactivate
```

---

## Step 3: Install Django

With your virtual environment activated, install Django using pip:

```bash
pip install django
```

### Verify Django Installation

Check the installed version:

```bash
django-admin --version
```

Expected output (version may vary):
```
5.0.6
```

View all installed packages:

```bash
pip list
```

---

## Step 4: Create a Django Project

Create a new Django project named `portfolio`:

```bash
django-admin startproject portfolio
cd portfolio
```

### Project Structure

After creation, your project structure should look like this:

```bash
tree
```

```
.
├── manage.py
└── portfolio
    ├── __init__.py
    ├── asgi.py
    ├── settings.py
    ├── urls.py
    └── wsgi.py
```

| File | Purpose |
|------|---------|
| `manage.py` | Command-line utility for Django administration |
| `__init__.py` | Empty file indicating this is a Python package |
| `settings.py` | Project configuration (apps, databases, middleware, etc.) |
| `urls.py` | URL declarations (routing configuration) |
| `asgi.py` | ASGI configuration for asynchronous serving |
| `wsgi.py` | WSGI configuration for deployment |

## Step 6: Create Your First App

In Django, projects contain one or more apps. Create an app named `home`:

```bash
python manage.py startapp home
```

### Updated Project Structure

```bash
tree
```

```
.
├── home
│   ├── __init__.py
│   ├── admin.py
│   ├── apps.py
│   ├── migrations
│   │   └── __init__.py
│   ├── models.py
│   ├── tests.py
│   └── views.py
├── manage.py
└── portfolio
    ├── __init__.py
    ├── asgi.py
    ├── settings.py
    ├── urls.py
    └── wsgi.py
```

### Register the App

Edit `portfolio/settings.py` and add `'home'` to the `INSTALLED_APPS` list:

```python
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'home',  # Add this line
]
```

---

## Step 7: Create a Simple View

Edit `home/views.py` to create a basic view:

```python
from django.http import HttpResponse

def home(request):
    return HttpResponse("Hello, Django!")
```

### What This Does

- `from django.http import HttpResponse` - Imports the HTTP response class
- `def home(request):` - Defines a view function that takes a request object
- `return HttpResponse("Hello, Django!")` - Returns a simple text response

---

## Step 8: Configure URL Routing

Edit `portfolio/urls.py` to map the root URL to your new view:

```python
from django.contrib import admin
from django.urls import path
from home.views import home

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', home),  # Maps the root URL to the home view
]
```

### URL Pattern Breakdown

| Pattern | View | Description |
|---------|------|-------------|
| `admin/` | `admin.site.urls` | Django's built-in admin interface |
| `''` (empty) | `home` | Root URL of your website |

Django uses SQLite by default (no additional configuration needed). Run migrations to create the necessary database tables:

```bash
python manage.py migrate
```

Expected output:
```

  Applying contenttypes.0001_initial... OK
  Applying auth.0001_initial... OK
  Applying admin.0001_initial... OK
```

### Database File

Create a superuser to access the Django admin interface:

```bash

```

Follow the prompts:

```bash

Password: ********

```

### Access Admin Panel

Start the server:

```bash
python manage.py runserver
```

Navigate to: `http://127.0.0.1:8000/admin`

Log in with the credentials you just created.

## Step 12: Use Templates (HTML)

Create a `templates` folder inside the `home` app:

```bash
mkdir home/templates
```

### Create HTML Template File

Create `home/templates/home.html`:

```bash
nano home/templates/home.html
```

Add the following HTML:

```html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Django App</title>
    <style>
        body {
            font-family: Arial, sans-serif;
            margin: 0;
            padding: 0;
            background: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
            min-height: 100vh;
            display: flex;
            justify-content: center;
            align-items: center;
        }
        .container {
            background: white;
            padding: 2rem;
            border-radius: 10px;
            box-shadow: 0 10px 25px rgba(0,0,0,0.1);
            text-align: center;
        }
        h1 {
            color: #333;
            margin-bottom: 0.5rem;
        }
        p {
            color: #666;
        }
    </style>
</head>
<body>
    <div class="container">
        <h1>Welcome to Django App</h1>
        <p>Your Django installation is working correctly!</p>
        <p>🎉 Congratulations! 🎉</p>
    </div>
</body>
</html>
```

### Update Views.py

Edit `home/views.py` to use the template instead of the direct HTTP response:

```python
from django.shortcuts import render

def home(request):
    return render(request, 'home.html')
```

> **Note:** You no longer need to import `HttpResponse` when using `render()`.

### Run the Server

```bash
python manage.py runserver
```

Visit `http://127.0.0.1:8000` - you should see your styled HTML page.

## Troubleshooting

### "command not found: django-admin"

**Cause:** Django not installed or virtual environment not activated.

**Solution:**
```bash
source venv/bin/activate
pip install django
```

### "Address already in use"

**Cause:** Another process is using port 8000.

**Solution:** Use a different port:
```bash
python manage.py runserver 8001
```

### Template not found

**Cause:** Django cannot locate your template file.

**Solution:** Ensure:
1. `templates` folder is in the correct location (`home/templates/`)
2. Template file name matches exactly (`home.html`)
3. App is registered in `INSTALLED_APPS`

### Migrations not applied

**Solution:**
```bash
python manage.py makemigrations
python manage.py migrate
```
