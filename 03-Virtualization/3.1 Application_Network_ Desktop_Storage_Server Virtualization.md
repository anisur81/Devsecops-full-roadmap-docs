## What is Virtualization?

Virtualization is the process of creating several virtual systems on a single server. This practice maximizes a physical machine's capacity by distributing its resources between multiple users and environments.

DevOps teams use virtualization to create **virtual machines (VMs)** , emulations of hardware and software configurations. Each VM has an operating system and acts as an independent computer even though it runs on a portion of the physical device. A virtual machine mimics all components of a computer, including:

- CPU
- RAM
- Storage
- Networks

With virtualization, a piece of hardware can host numerous VM configurations simultaneously without performance issues. The main benefits of virtualization are:

- More computing capabilities with fewer resources
- Running multiple independent systems on a single hardware
- Consistent environments throughout the continuous integration and continuous delivery (CI/CD) process

## Hypervisors

DevOps teams use a **hypervisor** to manage virtual machines. A hypervisor, or Virtual Machine Monitor (VMM), is a piece of software, firmware, or hardware that creates and runs VMs. The most popular VMMs are:

- VMWare (ESXi)
- AWS XEN
- Microsoft Hyper V
- AntsleOS
- Oracle VM Server
- POWER Hypervisor

The hypervisor runs on top of a physical dedicated server or an operating system to emulate the underlying hardware.

Virtualization relies on cloud computing to ensure optimal performance at all times. Cloud allows a VM to scale up or down on-demand and in a matter of minutes to meet resource requirements.

## Role of Virtualization in DevOps

Virtualization plays a vital role in DevOps. It automates various software development processes, including testing and delivery. With the help of virtualization, IT teams can develop and test within virtual and simulated environments using similar devices and systems to the end-users. This way, development and testing become more efficient and less time-consuming.

Virtual live environments can also be provided to test software at the deployment level. This helps in real-time testing, as the team can check the effect of every new change made to the software. By doing these tasks in virtualized environments, the amount of computing resources is reduced. This real-time testing helps in increasing the quality of the product.

Working with a virtual environment reduces the time for retesting and rebuilding the software for production. Thus, it reduces the extra efforts for the DevOps team, while ensuring faster and reliable delivery.

## How Does Virtualization Help DevOps?

Virtualization is a vital part of the DevOps software stack. Virtual machines allow a team to build, test, and deploy code within simulated environments without wasting computing resources. The benefits of virtualization include:

- More agility, flexibility, and scalability during development
- Cost savings across the SDLC, primarily in terms of maintenance and testing
- Faster workloads and environment setups make the team more productive

Virtualization in DevOps is essential in the development of complex cloud, API, and SOA systems. VMs are ideal for test-driven development (TDD) teams that prefer to start their bug hunt at the API layer.

DevOps teams often use virtualization alongside **containerization** to add more flexibility to software creation. Containerization tools allow engineers to run multiple applications within a single container, an environment that contains everything an app needs to run:

- Code
- System tools
- Libraries

The use of both VMs and containers supports complex application infrastructures. For example, a platform's front-end and middleware components might run in a container while the back-end components operate on a VM.

## Challenges of Virtualization

Despite having several perks, virtualization also has some challenges or limitations:

### 1. Time Consumption
Even if development and testing time is saved, virtualization still consumes much time, as its configuration and application need time.

### 2. Security Risk
There is a big chance of data breach with virtualization, as remote accessibility and virtualizing desktops or applications is not a very secure option.

### 3. Infrastructure Knowledge
To work with virtualization, the IT staff must have expertise. Hence, either existing employees must be trained, or new employees are required. This involves much time and costs much money.

# Different Types of Virtualizations

There are several primary types of virtualization:

- Server virtualization
- Network virtualization
- Desktop virtualization
- Operating system virtualization
- Storage virtualization
- Application virtualization

## Server Virtualization

Server virtualization enables a single physical server to perform multiple independent functions. This form of virtualization leads to:

- Reduced operating costs
- Increased server performance
- Faster workload capacity

Server virtualization also lowers managing costs and physical server complexity.

## Network Virtualization

Network virtualization mimics a network in a virtual environment. This virtualization process distributes network functions (directory services, file sharing, IP configurations) among virtual environments.

Network virtualization reduces the number of physical components necessary for setting up multiple separate networks, such as:

- Switches
- Routers
- Servers
- Cables
- Hubs

Virtual networks also offer better operational capabilities than their physical counterparts and do so at a reduced cost.

# Desktop Virtualization

Simply speaking, desktop virtualization refers to creating a replica or version of static software on the end user's desktop or operating system. This virtualization desktop helps the end user access their static desktop remotely anytime and anywhere.

Desktop virtualization is a method of creating a workstation with all the apps and widgets your employees normally use so that they can access their desktop from any device or location. The technology allows creating and storing multiple user workstations on a single host, residing in a data center or the cloud. The only condition is a stable Internet or network connection.

## Types of Desktop Virtualization

### Virtual Desktop Infrastructure (VDI)

If you opt for VDI, you will get a specific virtual machine assigned to each of your users with an operating system and all required system resources (device drivers, memory, etc.). It will be powered by a purpose-specific software (hypervisor). You can either invest in your own server equipment and operate in-house VDI or turn to a service provider.

Once the connection with a server is established, a user gets a desktop image transferred in real-time over the network and can interact with that virtual image the same way they'd operate a local computer.

### Desktop-as-a-Service (DaaS)

In a DaaS deployment model, you basically rent cloud-based virtual machines from a service provider. This option is much less customizable than VDI. In most cases, you will get one typical virtual desktop for all your employees with a limited number of software products installed and no personal settings stored.

On the bright side, DaaS is a fast, easy-scalable solution that allows you to never pay for features or services you don't need. Plus, your virtual desktops will be accessible from any place in the world and any internet-connected device.

### Remote Desktop Services (RDS)

RDS (a.k.a. Microsoft Terminal Server) is very similar to VDI but is exclusively powered by Microsoft Windows Server operating system and works via Microsoft Remote Desktop Protocol (RDP). Because of this, you will be limited to Windows Server apps for your virtual desktops. To get more than two users connected, you'll need to purchase additional licenses.

## How to Choose the Right Desktop Virtualization Type

Consider these factors:

| Factor | Consideration |
|--------|----------------|
| **Budget** | Balance between cost savings and infrastructure control |
| **Competence** | Ensure IT team can manage in-house solutions |
| **Safety** | Outsourcing may not work for strict data security policies |
| **Control** | Cloud solutions give away some infrastructure control |
| **Location** | Distance to data center affects latency |
| **Scalability** | Time to add new team members may be critical |

## Benefits of Desktop Virtualization

- ✅ **Optimal resource management** - No need for expensive high-end devices for every employee
- ✅ **Budget savings** - Fraction of the cost of physical servers and workstations
- ✅ **Improved flexibility** - Work anytime, anywhere, from any device
- ✅ **No hardware constraints** - Run resource-demanding software on cheap machines
- ✅ **Instant updates** - Updates apply once on the server for all users
- ✅ **Extra security** - Multi-level authentication and access control
- ✅ **Improved end-user experience** - Access to printing and USB devices
- ✅ **BYOD support** - No corporate data stored on personal devices

## Disadvantages of Desktop Virtualization

- ❎ **Initial investments** - Significant cost for in-house deployment
- ❎ **Bandwidth requirements** - Network must handle extra load
- ❎ **Availability issues** - Performance depends on Internet provider reliability

# Operating System Virtualization

Operating system virtualization allows a developer to deploy multiple operating systems on a single machine. This virtualization type helps a team to:

- Reduce bulk hardware costs
- Test an app in multiple operating systems on a single machine
- Increase security due to isolation of virtual instances
- Speed up IT services

Operating system virtualization is a popular option when a team needs to run Linux and Windows environments on a single device.

## How OS-Based Virtualization Works

The host operating system has a feature that allows it to create and manage containers. This feature is called the **kernel**. The kernel controls resource access and allocation to each container and isolates containers from each other and from the host system.

The kernel also provides resource management features to limit the impact of one container's activities on other containers (CPU time, memory, disk space, network bandwidth).

## OS-Based Services

- **Backup and recovery** - Back up and restore virtual machines
- **Security management** - Configure firewall, antivirus, and other settings
- **Integration to directory services** - Centrally manage users and groups

## OS-Based Operations

- **Hardware capabilities** - Access network, CPU, memory, and storage
- **Connected peripherals** - Access webcam, printer, keyboard, scanner
- **Data** - Read/write to files, folders, and network shares

## Advantages of OS-Based Virtualization

- Resource isolation ensures independent and secure container operation
- Containers are lightweight and save disk space
- Better performance by directly accessing hardware resources
- Containers scale well across systems
- Portability across different computers

## Limitations of OS-Based Virtualization

- Compatibility issues with certain applications and devices
- Some apps need direct hardware access not supported by the host OS
- Security relies on host OS stability
- Complexity requires skills to set up and manage multiple containers

# Storage Virtualization in Cloud Computing

Storage virtualization in cloud computing is a technology that abstracts physical storage resources from applications and users, presenting them as a single, unified virtual storage pool. This allows multiple storage devices to be managed as a single entity, providing more flexibility, scalability, and ease of management.

## Importance of Storage Virtualization

| Benefit | Description |
|---------|-------------|
| **Promptness** | Quick allocation without manual intervention |
| **Optimal resource utilization** | More efficient use of hardware, lower TCO |
| **Scalability** | Seamless growth without service disruptions |
| **Data migration & mobility** | Simplified data movement between arrays |
| **Centralized management** | Simplified administration tasks |

## Types of Storage Virtualization

### 1. Host-Based Storage Virtualization
Performed at the host level. Software installed on servers manages local storage resources. Adaptable and simple, but may have performance overhead.

### 2. Array-Based Storage Virtualization
Storage arrays manage pooling of resources. Offers centralized management and excellent performance.

### 3. Network-Based Storage Virtualization
Uses a dedicated storage virtualization appliance as an intermediary. Popular for large enterprises with complex storage infrastructures.

## How Storage Virtualization Works

Physical storage hardware is mirrored in a virtual volume. Multiple physical disks are combined into a grouping using a single server. A virtualization layer separates physical disks from the virtual volume, enabling operating systems and applications to access storage.

Physical disks are divided into small blocks of data known as:
- Logical unit numbers (LUNs)
- Logical volumes (LV)
- RAID groups

## Benefits of Storage Virtualization

- **Lower cost** - Fewer hardware redundancies needed
- **Saved time** - Reduced downtime, less disruptive upgrades
- **Scalability** - Dynamic provisioning on demand
- **Easier management** - Simple addition/deletion without interrupting apps
- **Reduced risk** - Mirrored components reduce disruption risk
- **Increased productivity** - 100% availability, faster deployments
- **Improved efficiency** - Better capacity utilization, little chance of running out of capacity

## Additional Storage Virtualization Types

### Block-Level Storage Virtualization
Abstracts storage at the block level. Used in SANs for high performance and flexibility.

### File-Level Storage Virtualization
Manages data at the file system level. Used in NAS environments.

### Object-Level Storage Virtualization
Handles data as objects with metadata. Popular in cloud storage for unstructured data.

### Hypervisor-Based Storage Virtualization
Integrates storage management within the hypervisor. Used in VMware or Hyper-V environments.

### Thin Provisioning Storage Virtualization
Allocates virtual capacity that appears larger than physical storage. Optimizes storage use.

# What is Application Virtualization?

Application virtualization is a technology that encapsulates an application and its dependencies into a self-contained package, isolating it from the underlying operating system, and allowing it to run in a controlled, virtualized environment.

In traditional software deployment, applications are installed directly on the host operating system, often leading to compatibility issues. Application virtualization solves this by encapsulating each application in a container, ensuring applications can coexist peacefully.

## Application Virtualization Examples

| Example | Description |
|---------|-------------|
| **Thin Clients & RDS** | Lightweight terminals run applications from central server |
| **Server Virtualization** | Encapsulates applications on virtual servers |
| **VDI** | Virtual desktops with applications on servers |
| **SaaS** | Google Workspace, Microsoft 365 delivered virtually |
| **Legacy Application Support** | Run old apps on modern hardware/OS |
| **Containers** | Docker encapsulates apps for portability |

## Application Virtualization Benefits

- **Isolation and Compatibility** - Applications don't interfere with each other
- **Simplified Deployment** - Package as standalone virtualized instances
- **Enhanced Security** - Malware cannot easily infect host system
- **Portability** - Run on different machines and OS
- **Efficient Resource Utilization** - Improved system stability and performance

## Disadvantages of Application Virtualization

- **Performance Overhead** - Some latency, especially for graphics-intensive apps
- **Complexity** - Requires understanding and skilled personnel
- **Licensing and Costs** - Potential licensing costs and hardware investment
- **Limited Application Support** - Some complex software may not work well

## Types of Application Virtualization

### Full Virtualization
Entire operating system is encapsulated. Used in hypervisor-based solutions (VMware, Hyper-V).

### Partial Virtualization
Only specific components are isolated. More lightweight than full virtualization.

### Application Streaming
Provides portions of an application on-demand over a network. Optimizes bandwidth and storage.

### Containerization
Lightweight, efficient approach (Docker). Shares kernel with host system. Popular in DevOps and cloud computing.
