# Managing Container Stdout & Stderr Logs in Kubernetes

## Basic Log Commands

### Single Container Pod
Return snapshot logs from pod nginx with only one container:
```bash
kubectl logs nginx
```

### Multi-Container Pod
Return snapshot logs from pod nginx with multi containers:
```bash
kubectl logs nginx --all-containers=true
```

### Label Selection
Return snapshot logs from all containers in pods defined by label app=nginx:
```bash
kubectl logs -l app=nginx --all-containers=true
```

## Advanced Log Retrieval

### Previous Terminated Container
Return snapshot of previous terminated ruby container logs from pod web-1:
```bash
kubectl logs -p -c ruby web-1
```

### Streaming Logs
Begin streaming the logs of the ruby container in pod web-1:
```bash
kubectl logs -f -c ruby web-1
```

Begin streaming the logs from all containers in pods defined by label app=nginx:
```bash
kubectl logs -f -l app=nginx --all-containers=true
```

## Log Filtering and Limiting

### Limit Output Lines
Display only the most recent 20 lines of output in pod nginx:
```bash
kubectl logs --tail=20 nginx
```

### Time-Based Filtering
Show all logs from pod nginx written in the last hour:
```bash
kubectl logs --since=1h nginx
```

### Insecure TLS (for Kubelet)
Show logs from a kubelet with an expired serving certificate:
```bash
kubectl logs --insecure-skip-tls-verify-backend nginx
```

## Working with Workload Resources

### Job Logs
Return snapshot logs from first container of a job named hello:
```bash
kubectl logs job/hello
```

### Deployment Logs
Return snapshot logs from container nginx-1 of a deployment named nginx:
```bash
kubectl logs deployment/nginx -c nginx-1
```

## Key Options Summary

| Option | Description |
|--------|-------------|
| `-c, --container` | Specify container name (required for multi-container pods) |
| `-f, --follow` | Stream logs continuously |
| `--all-containers` | Get logs from all containers in pod |
| `--tail` | Display only last N lines |
| `--since` | Show logs since a specific duration (e.g., `1h`, `30m`) |
| `-p, --previous` | Show logs from previous terminated container |
| `-l, --selector` | Filter pods by label selector |
| `--insecure-skip-tls-verify-backend` | Skip TLS verification (for testing) |

## Time Duration Formats

| Format | Description |
|--------|-------------|
| `1s` | 1 second |
| `1m` | 1 minute |
| `1h` | 1 hour |
| `1d` | 1 day |
| `1h30m` | 1 hour and 30 minutes |

## Common Use Cases

### Troubleshooting a Failing Pod
```bash
# Check recent logs from the last 5 minutes
kubectl logs --since=5m -c ruby web-1

# Check previous container logs if pod is restarting
kubectl logs -p -c ruby web-1
```

### Following Multiple Containers
```bash
# Stream logs from all containers in a specific pod
kubectl logs -f --all-containers=true nginx

# Stream logs from all pods with a specific label
kubectl logs -f -l app=nginx --all-containers=true
```

### Debugging Deployments
```bash
# Check logs from a specific pod in a deployment
kubectl logs deployment/nginx -c nginx-1

# Check logs from all pods in a deployment (using label selector)
kubectl logs -l app=nginx --all-containers=true
```

## Best Practices

1. **Use `--tail` with `-f`** to limit initial output when streaming:
   ```bash
   kubectl logs -f --tail=50 nginx
   ```

2. **Combine filters** for precise log retrieval:
   ```bash
   kubectl logs --since=10m --tail=20 nginx
   ```

3. **Always specify container** in multi-container pods:
   ```bash
   kubectl logs pod-name -c container-name
   ```

4. **Use labels** for managing logs of related pods:
   ```bash
   kubectl logs -l app=nginx,environment=production
   ```

5. **Check previous logs** when investigating pod restarts:
   ```bash
   kubectl logs -p pod-name -c container-name
   ```
