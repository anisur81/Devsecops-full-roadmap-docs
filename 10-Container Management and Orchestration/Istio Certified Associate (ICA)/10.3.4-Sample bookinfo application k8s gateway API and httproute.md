# Istio Gateway API Configuration Guide

## 1. Prerequisites

Before starting, ensure you have the following:

| Prerequisite | Description |
|--------------|-------------|
| Kubernetes Cluster | Running Kubernetes cluster |
| kubectl | Installed and configured |
| MetalLB | Installed and configured |
| Istio | Installed with demo profile |
| Gateway API CRDs | Installed |
| Internet Access | To pull container images |

---

## 2. Target Architecture

```
Internet
   ↓
MetalLB External IP
   ↓
Istio IngressGateway Controller
   ↓
Kubernetes Gateway API Gateway
   ↓
HTTPRoute
   ↓
Bookinfo Services
```

---

## 5. Create Kubernetes Gateway

### Gateway Resource Definition

**File:** `gateway-api.yaml`

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: Gateway
metadata:
  name: bookinfo-gateway
  namespace: gateway-system
spec:
  gatewayClassName: istio
  listeners:
  - name: http
    protocol: HTTP
    port: 80
    allowedRoutes:
      namespaces:
        from: All
```

### Apply Gateway Configuration

```bash
kubectl apply -f gateway-api.yaml
```

---

## 6. Create HTTPRoute

### HTTPRoute Resource Definition

**File:** `httproute.yaml`

```yaml
apiVersion: gateway.networking.k8s.io/v1
kind: HTTPRoute
metadata:
  name: bookinfo-route
  namespace: bookinfo
spec:
  parentRefs:
  - name: bookinfo-gateway
    namespace: gateway-system
  rules:
  - matches:
    - path:
        type: Exact
        value: /productpage
    - path:
        type: PathPrefix
        value: /static
    - path:
        type: Exact
        value: /login
    - path:
        type: Exact
        value: /logout
    - path:
        type: PathPrefix
        value: /api/v1/products
    backendRefs:
    - name: productpage
      namespace: bookinfo
      port: 9080
```

### Apply HTTPRoute Configuration

```bash
kubectl apply -f httproute.yaml
```

## 12. Traffic Flow Explanation

1. **User** sends request from browser or curl
2. **Request** reaches MetalLB external IP
3. **Gateway API Gateway** receives the request
4. **HTTPRoute** evaluates routing rules
5. **Traffic** forwards to productpage Kubernetes service
6. **Service** routes traffic to Bookinfo application pods
7. **Response** returns to the client

## 14. Troubleshooting

### Common Issues and Solutions

#### Issue: MetalLB external IP not assigned
```bash
# Check MetalLB status
kubectl get pods -n metallb-system

# Check IP address pool
kubectl get ipaddresspool -n metallb-system
```

#### Issue: Istio ingress gateway pods not running
```bash
# Check ingress gateway status
kubectl get pods -n istio-system

# Check ingress gateway logs
kubectl logs -n istio-system deploy/istio-ingressgateway
```

#### Issue: Gateway or HTTPRoute status not programmed
```bash
# Describe Gateway for detailed status
kubectl describe gateway bookinfo-gateway -n gateway-system

# Describe HTTPRoute for detailed status
kubectl describe httproute bookinfo-route -n bookinfo
```

#### Issue: Bookinfo pods not healthy
```bash
# Check Bookinfo pods
kubectl get pods -n bookinfo

# Check pod logs
kubectl logs -n bookinfo <pod-name>
```

#### Issue: Sidecar injection not enabled
```bash
# Enable sidecar injection in bookinfo namespace
kubectl label namespace bookinfo istio-injection=enabled

# Verify namespace has correct label
kubectl get namespace -L istio-injection
```

#### Issue: Routing fails
```bash
# Inspect Istio proxy logs
kubectl logs -n bookinfo productpage-v1-<pod-suffix> -c istio-proxy

# Check route configuration
istioctl proxy-config routes deploy/istio-ingressgateway -n istio-system
```

## 15. Additional Commands

### Check Gateway Status with Detailed Info

```bash
# Get detailed Gateway information
kubectl get gateway bookinfo-gateway -n gateway-system -o yaml

# Check Gateway conditions
kubectl get gateway bookinfo-gateway -n gateway-system -o json | jq '.status'
```

### Check HTTPRoute Status

```bash
# Get detailed HTTPRoute information
kubectl get httproute bookinfo-route -n bookinfo -o yaml

# Check HTTPRoute conditions
kubectl get httproute bookinfo-route -n bookinfo -o json | jq '.status'
```

### Verify IngressGateway Service

```bash
# Check LoadBalancer service
kubectl get svc istio-ingressgateway -n istio-system

# Check endpoints
kubectl get endpoints istio-ingressgateway -n istio-system
```

### Test Application with curl

```bash
# Test productpage endpoint
curl http://172.18.0.60/productpage

# Check if routing works for static files
curl http://172.18.0.60/static/css/style.css

# Test API endpoint
curl http://172.18.0.60/api/v1/products
```

## 16. Cleanup

To remove Gateway API resources:

```bash
# Delete HTTPRoute
kubectl delete httproute bookinfo-route -n bookinfo

# Delete Gateway
kubectl delete gateway bookinfo-gateway -n gateway-system
```
