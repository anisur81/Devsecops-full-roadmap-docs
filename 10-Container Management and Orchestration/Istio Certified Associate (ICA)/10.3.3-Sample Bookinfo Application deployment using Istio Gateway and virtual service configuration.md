# Istio Gateway Configuration Guide

## 1. Create Istio Gateway

### Gateway Resource Definition

**File:** `istiogateway.yaml`

```yaml
apiVersion: networking.istio.io/v1
kind: Gateway
metadata:
  name: bookinfo-gateway
spec:
  selector:
    istio: ingressgateway
  servers:
  - port:
      number: 80
      name: http
      protocol: HTTP
    hosts:
    - "*"
```

### Apply Gateway Configuration

```bash
kubectl apply -f istiogateway.yaml
```

### Verify Gateway

```bash
kubectl get gateways.networking.istio.io -n default
```

**Expected output:**
```
NAME               AGE
bookinfo-gateway   17h
```

---

## 2. Create Virtual Service

### VirtualService Resource Definition

**File:** `vsvc.yaml`

```yaml
apiVersion: networking.istio.io/v1
kind: VirtualService
metadata:
  name: bookinfo
spec:
  hosts:
  - "*"
  gateways:
  - bookinfo-gateway
  http:
  - match:
    - uri:
        exact: /productpage
    - uri:
        prefix: /static
    - uri:
        exact: /login
    - uri:
        exact: /logout
    - uri:
        prefix: /api/v1/products
    route:
    - destination:
        host: productpage
        port:
          number: 9080
```

### Apply VirtualService Configuration

```bash
kubectl apply -f vsvc.yaml
```

### Verify VirtualService

```bash
kubectl get virtualservice -n default
```

**Expected output:**
```
NAME       GATEWAYS               HOSTS   AGE
bookinfo   ["bookinfo-gateway"]   ["*"]   17h
```

## 4. Traffic Flow Architecture

```
Internet (Client user)
   ↓
istio-ingressgateway (LoadBalancer)
   ↓
Gateway (Istio rule)
   ↓
VirtualService (routing logic)
   ↓
productpage → reviews → details
```

## 10. Troubleshooting

### Issue: 404 Not Found
```bash
# Verify VirtualService routing rules
istioctl proxy-config routes deploy/istio-ingressgateway -n istio-system

# Check if the service exists
kubectl get svc productpage -n default

# Verify the gateway selector matches ingressgateway labels
kubectl describe gateway bookinfo-gateway
```

### Issue: No External IP
```bash
# Check if LoadBalancer is pending
kubectl get svc istio-ingressgateway -n istio-system -w

# For minikube, enable minikube tunnel
minikube tunnel
```

### Issue: Routes Not Loading
```bash
# Restart ingressgateway
kubectl rollout restart deploy/istio-ingressgateway -n istio-system

# Check gateway events
kubectl describe gateway bookinfo-gateway
```

## 11. Advanced Configuration

### Add HTTPS Support

To add HTTPS, modify the Gateway:

```yaml
apiVersion: networking.istio.io/v1
kind: Gateway
metadata:
  name: bookinfo-gateway
spec:
  selector:
    istio: ingressgateway
  servers:
  - port:
      number: 443
      name: https
      protocol: HTTPS
    tls:
      mode: SIMPLE
      credentialName: bookinfo-credential
    hosts:
    - "*"
  - port:
      number: 80
      name: http
      protocol: HTTP
    hosts:
    - "*"
```

### Add Multiple Hosts

```yaml
apiVersion: networking.istio.io/v1
kind: VirtualService
metadata:
  name: bookinfo
spec:
  hosts:
  - "bookinfo.example.com"
  - "bookinfo.dev.example.com"
  gateways:
  - bookinfo-gateway
  http:
  - match:
    - uri:
        exact: /productpage
    route:
    - destination:
        host: productpage
        port:
          number: 9080
```

## 12. Cleanup

To remove Gateway and VirtualService:

```bash

kubectl delete virtualservice bookinfo -n default

# Delete Gateway
kubectl delete gateway bookinfo-gateway -n default
```
