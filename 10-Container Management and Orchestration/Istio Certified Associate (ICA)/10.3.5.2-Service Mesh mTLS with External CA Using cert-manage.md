## 1. Architecture Overview

```
Root CA (offline)
      ↓
Intermediate CA (cert-manager secret)
      ↓
ClusterIssuer
      ↓
cert-manager-istio-csr
      ↓
Istio requests CSR (ISTIO with cert-manager-istio-csr)
      ↓
cert-manager signs via Intermediate CA
      ↓
Workload certificates issued
      ↓
mTLS STRICT mode active
```

## 3. Upload Intermediate CA to Kubernetes

### Step 3.1: Create Secret from Intermediate CA

Transfer the `intermediate.crt.pem` and `intermediate.key.pem` files from the ROOT CA server to the Kubernetes node.

```bash
kubectl create secret tls inter-ca-secret \
  --cert=intermediate.crt.pem \
  --key=intermediate.key.pem \
  -n cert-manager
```

### Step 3.2: Verify Secret

```bash
kubectl get secret -n cert-manager
```

**Expected output:**
```
NAME               TYPE                DATA   AGE
inter-ca-secret    kubernetes.io/tls   2      10s
```

### Step 3.3: Extract Certificate for Verification

```bash
# Extract CA certificate
kubectl get secret inter-ca-secret -n cert-manager -o jsonpath='{.data.tls\.crt}' | base64 -d > ca-cert.pem

# Extract CA private key
kubectl get secret inter-ca-secret -n cert-manager -o jsonpath='{.data.tls\.key}' | base64 -d > ca-key.pem
```

## 4. Create ClusterIssuer

### Step 4.1: Create ClusterIssuer Configuration

**File:** `root-ca-issuer.yaml`

```yaml
apiVersion: cert-manager.io/v1
kind: ClusterIssuer
metadata:
  name: root-ca-issuer
  namespace: cert-manager
spec:
  ca:
    secretName: inter-ca-secret
```

### Step 4.2: Apply ClusterIssuer

```bash
kubectl apply -f root-ca-issuer.yaml
```

### Step 4.3: Verify ClusterIssuer

```bash
kubectl get clusterissuer
```

**Expected output:**
```
NAME             READY   AGE
root-ca-issuer   True    10s
```

## 5. Install Helm

### Step 5.1: Add Helm Repository


## 6. Install cert-manager-istio-csr

### Step 6.1: Add Jetstack Helm Repository

```bash
helm repo add jetstack https://charts.jetstack.io
helm repo update
```

### Step 6.2: Install cert-manager-istio-csr

```bash
helm install cert-manager-istio-csr jetstack/cert-manager-istio-csr \
  --namespace cert-manager \
  --set app.certmanager.issuer.name=root-ca-issuer \
  --set app.certmanager.issuer.kind=ClusterIssuer \
  --set app.certmanager.issuer.group=cert-manager.io \
  --set app.tls.trustDomain=cluster.local \
  --set app.rootCAFile=ca.crt
```

**Expected output:**
```
NAME: cert-manager-istio-csr
LAST DEPLOYED: Thu Jun  4 18:13:45 2026
NAMESPACE: cert-manager
STATUS: deployed
REVISION: 1
TEST SUITE: None
```

## 10. Create mTLS PeerAuthentication

### Step 10.1: Create PeerAuthentication Configuration

**File:** `mesh-mtls.yaml`

```yaml
apiVersion: security.istio.io/v1
kind: PeerAuthentication
metadata:
  name: mtls-csr
  namespace: bookinfo
spec:
  mtls:
    mode: STRICT
```

### Step 10.2: Apply PeerAuthentication

```bash
kubectl apply -f mesh-mtls.yaml
```

### Step 10.3: Verify PeerAuthentication

```bash
kubectl get PeerAuthentication -A
```

**Expected output:**
```
NAMESPACE   NAME      MODE     AGE
bookinfo    mtls-csr  STRICT   10s
```

## 11. Deploy Test Application

### Step 11.1: Create Deployment

```bash
kubectl create deployment nginx --image=nginx -n bookinfo
```

### Step 11.2: Create Gateway Configuration

**File:** `nginx-istio-gw.yaml`

```yaml
apiVersion: networking.istio.io/v1
kind: Gateway
metadata:
  name: nginx-gateway
  namespace: bookinfo
spec:
  selector:
    istio: ingressgateway
  servers:
  - port:
      number: 80
      name: http
      protocol: HTTP
    hosts:
    - "*"
```

### Step 11.3: Create VirtualService

**File:** `nginx-vsvc.yaml`

```yaml
apiVersion: networking.istio.io/v1
kind: VirtualService
metadata:
  name: nginx-vs
  namespace: bookinfo
spec:
  hosts:
  - "*"
  gateways:
  - nginx-gateway
  http:
  - route:
    - destination:
        host: nginx.bookinfo.svc.cluster.local
        port:
          number: 80
```

### Step 11.4: Apply Configurations

```bash
kubectl apply -f nginx-istio-gw.yaml -n bookinfo
kubectl apply -f nginx-vsvc.yaml -n bookinfo
```

### Step 11.5: Create Test Client

```bash
kubectl run curl-client -n bookinfo \
  --image=curlimages/curl \
  --restart=Never \
  -- sleep 3600
```

### Step 11.6: Test Connectivity

```bash
kubectl exec -it curl-client -n bookinfo -- sh
```

Inside the container:
```bash
curl http://nginx.bookinfo.svc.cluster.local
```

## 14. Troubleshooting

### Issue: cert-manager-istio-csr Not Running

```bash
# Check logs
kubectl logs -n cert-manager -l app=cert-manager-istio-csr

# Verify ClusterIssuer status
kubectl describe clusterissuer root-ca-issuer
```

### Issue: Certificates Not Issued

```bash
# Check certificate requests
kubectl get certificaterequests -A

# Check istiod logs
kubectl logs -n istio-system deploy/istiod --tail=50 | grep -i certificate
```

### Issue: mTLS Connection Failed

```bash
# Check peer authentication status
kubectl get peerauthentication -A

# Verify sidecar injection
kubectl get pods -n bookinfo -o yaml | grep -A 5 istio-proxy

# Check proxy logs
kubectl logs -n bookinfo nginx-xxx -c istio-proxy
```

### Issue: CSR Not Signed

```bash
# Check cert-manager logs
kubectl logs -n cert-manager deployment/cert-manager

# Verify CSR agent is running
kubectl get pods -n cert-manager -l app=cert-manager-istio-csr
```
