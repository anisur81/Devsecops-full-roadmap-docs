
# Firewall Ports Documentation

## Ports 22, 80, and 443

---

# 1. Introduction

This document provides detailed information about commonly used network ports:

* Port 22 (SSH)
* Port 80 (HTTP)
* Port 443 (HTTPS)

These ports are widely used in Linux servers, cloud environments, DevOps infrastructure, and web hosting systems.

---

# 2. Firewall Overview

A firewall is a security system that monitors and controls incoming and outgoing network traffic based on predefined security rules.

## Main Functions

* Allow authorized traffic
* Block unauthorized access
* Protect servers and applications
* Monitor network communication

---

# 3. Port 22 – SSH (Secure Shell)

## 3.1 Description

Port 22 is used for SSH (Secure Shell), a secure protocol for remote server management.

## 3.2 Protocol

* TCP

## 3.3 Main Uses

* Remote Linux server access
* Secure command execution
* File transfer using SCP/SFTP
* Remote administration

## 3.4 Example

```bash
ssh username@server-ip
```

## 3.5 Security Risks

* Brute-force attacks
* Unauthorized login attempts
* Credential theft

## 3.6 Security Best Practices

* Use SSH key authentication
* Disable root login
* Restrict access by IP address
* Enable Fail2Ban
* Use strong passwords

## 3.7 Firewall Configuration

### UFW

```bash
sudo ufw allow 22/tcp
```

### iptables

```bash
sudo iptables -A INPUT -p tcp --dport 22 -j ACCEPT
```

---

# 4. Port 80 – HTTP

## 4.1 Description

Port 80 is used for HTTP (HyperText Transfer Protocol), which handles normal web traffic without encryption.

## 4.2 Protocol

* TCP

## 4.3 Main Uses

* Website hosting
* Web applications
* API communication

## 4.4 Example

```text
http://example.com
```

## 4.5 Security Risks

* Unencrypted data transmission
* Man-in-the-middle attacks
* Packet sniffing

## 4.6 Best Practices

* Redirect HTTP traffic to HTTPS
* Avoid transmitting sensitive information over HTTP

## 4.7 Firewall Configuration

### UFW

```bash
sudo ufw allow 80/tcp
```

### iptables

```bash
sudo iptables -A INPUT -p tcp --dport 80 -j ACCEPT
```

---

# 5. Port 443 – HTTPS

## 5.1 Description

Port 443 is used for HTTPS (HTTP Secure), which encrypts communication using SSL/TLS certificates.

## 5.2 Protocol

* TCP

## 5.3 Main Uses

* Secure websites
* Online banking
* Login systems
* Secure APIs
* E-commerce platforms

## 5.4 Example

```text
https://example.com
```

## 5.5 Security Features

* Data encryption
* Secure authentication
* Data integrity protection

## 5.6 SSL/TLS Providers

* Let's Encrypt
* Cloudflare
* DigiCert

## 5.7 Firewall Configuration

### UFW

```bash
sudo ufw allow 443/tcp
```

### iptables

```bash
sudo iptables -A INPUT -p tcp --dport 443 -j ACCEPT
```

# 8. Checking Open Ports in Linux

## Using ss

```bash
sudo ss -tulpn
```

## Using netstat

```bash
sudo netstat -tulpn
```

## Using nmap

```bash
nmap localhost
```

# 9. Recommended Security Guidelines

* Keep unnecessary ports closed
* Restrict SSH access
* Use HTTPS instead of HTTP
* Regularly monitor firewall logs
* Update server security patches
* Use IDS/IPS systems where possible

---

# 10. Conclusion

Ports 22, 80, and 443 are essential for modern server infrastructure.

* Port 22 provides secure remote administration.
* Port 80 handles standard web traffic.
* Port 443 ensures encrypted and secure communication.

Proper firewall configuration and security practices are critical for protecting systems and applications from unauthorized access and cyber threats.
