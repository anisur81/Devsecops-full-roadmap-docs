# How To Configure Nginx as a Reverse Proxy on Ubuntu

## Prerequisites

To complete this tutorial, you will need:

- An **Ubuntu 24.04 LTS (or later)** server, set up according to the Initial Server Setup guide
- The **address of the application server** you want to proxy (referred to as `backend_server` throughout this tutorial)
- A **domain name** pointed at your server's public IP (will be configured with Nginx to proxy your application server)

---

## Step 1 — Installing Nginx

Nginx is available for installation with `apt` through the default repositories.

### Update and Install

```bash
sudo apt update
sudo apt install nginx
```

Press `Y` to confirm the installation.

### Verify Nginx is Running

```bash
systemctl status nginx
```

---

## Step 2 — Configuring Server Block

It is recommended practice to create a custom configuration file for your new server block additions, instead of editing the default configuration directly.

### Create a New Configuration File

Create and open a new Nginx configuration file using `nano` or your preferred text editor:

```bash
sudo nano /etc/nginx/sites-available/anis.example
```

### Example Configuration

```nginx
# Upstream Blocks for Load Balancing
upstream backend_servers {
    server 172.17.12.54:8050;
    server 172.17.12.54:8040;
    server 172.17.12.54:8030;
}

# Basic proxy_pass Configuration
# The proxy_pass directive is the foundation of NGINX reverse proxy configuration
# It tells NGINX where to forward incoming requests

server {
    listen 80;
    server_name app.anis.example;

    location / {
        proxy_pass http://backend_servers;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

### Enable the Configuration File

Create a link from the configuration file to the `sites-enabled` directory that Nginx reads at startup:

```bash
sudo ln -s /etc/nginx/sites-available/anis.example /etc/nginx/sites-enabled/
```

### List Enabled Sites

```bash
ls /etc/nginx/sites-enabled/
```

### Remove the Default Site

```bash
sudo rm /etc/nginx/sites-enabled/default
```

### Test Configuration for Syntax Errors

```bash
sudo nginx -t
```

### Restart Nginx to Apply Changes

```bash
sudo systemctl restart nginx
```

### Verify Nginx is Listening on Port 80

```bash
sudo ss -tulnp | grep :80
```

This command checks which process is listening on port 80 and whether Nginx is active.

---

## Step 3 — Testing the Reverse Proxy

Nginx is now configured as a reverse proxy for your application server. You can access it from a browser or using `curl` if your application server is running.

### Test with IP Address

```bash
curl http://172.17.12.77
```

### Test with Domain Name

```bash
curl http://app.anis.example
```

---

## Key Configuration Areas

Configuring NGINX as a reverse proxy involves understanding several key areas:

| Area | Description |
|------|-------------|
| **proxy_pass** | Forwards requests to backend servers |
| **upstream blocks** | Enable load balancing across multiple servers |
| **Headers** | Pass client information to backends |
| **Buffering** | Improves performance by freeing backends quickly |
| **Timeouts** | Prevent hung connections and handle slow backends |
| **Keepalive** | Reduces connection overhead for better performance |
| **Failover** | Ensures high availability when backends fail |

## Quick Reference

### Command Summary

| Command | Purpose |
|---------|---------|
| `sudo apt update` | Update repository index |
| `sudo apt install nginx` | Install Nginx |
| `systemctl status nginx` | Check Nginx service status |
| `sudo nginx -t` | Test configuration syntax |
| `sudo systemctl restart nginx` | Restart Nginx service |
| `sudo ss -tulnp \| grep :80` | Check process listening on port 80 |

### File Locations

| File/Directory | Purpose |
|----------------|---------|
| `/etc/nginx/sites-available/` | Store custom server block configurations |
| `/etc/nginx/sites-enabled/` | Enabled configurations (linked from sites-available) |

### Common proxy_set_header Directives

| Directive | Description |
|-----------|-------------|
| `proxy_set_header Host $host;` | Passes original host header |
| `proxy_set_header X-Real-IP $remote_addr;` | Passes client's real IP address |
| `proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;` | Passes proxy chain information |
| `proxy_set_header X-Forwarded-Proto $scheme;` | Passes original protocol (http/https) |
