# FTP vs SFTP

## What is FTP?

FTP (File Transfer Protocol) is a protocol that is used to transfer or copy files from one host to another host. However, there may be some problems like different file names and different file directories while sending and receiving files in different hosts or systems. 

In FTP, a secure channel is **not** provided to transfer files between hosts or systems. It uses **port no. 21**.

FTP is used for transferring web pages and downloading files from other different servers. Basically, it is used for transferring files from one system to another more reliably and efficiently.

---

## Advantages of FTP

- Speedy data transfer
- Transfer multiple files and directories
- Multitasking (transferring and downloading can go in parallel)
- Continuous transfer (transfers can be resumed from a given point)
- Many FTP clients have scripting features

---

## Disadvantages of FTP

- Few FTP providers do not offer encryption
- Possibility of eavesdropping because plain text is used for sensitive information such as passwords
- When sending files via FTP, compliance can be a problem

---

## What is SFTP?

SFTP (Secure File Transfer Protocol) is a protocol that provides a secure channel to transfer or copy files from one host to another host or system. SFTP establishes the control connection under the **SSH protocol** and uses **port no. 22**.

Basically, it is a protocol for transferring large files over the web. It can handle sensitive data securely in all situations. A user who has a business can successfully transmit files containing secret information by using SFTP.

---

## Advantages of SFTP

- Encryption
- Host authentication
- Compliance
- Only one connection is used (DATA connection is not required in SFTP)

---

## Disadvantages of SFTP

- SSH keys are difficult to maintain and verify
- Difficult to login because communication is binary
- The standards categorize some elements as optional or recommended, which causes some incompatibilities between software products from various vendors
- Slow speed
- Communication is not human-readable as it is in binary form

---

## Differences Between FTP and SFTP

| Feature | FTP | SFTP |
|---------|-----|------|
| **Full Form** | File Transfer Protocol | Secure File Transfer Protocol |
| **Developed By** | Abhay Bhushan (published as RFC in 1971) | Sami Lehtinen assisted Tatu Ylönen (1997) |
| **Secure Channel** | Not provided | Provided |
| **Protocol Family** | Part of TCP/IP protocol | SSH protocol |
| **Default Port** | Port 21 | Port 22 |
| **Connection Type** | Establishes connection under TCP protocol | Establishes control connection under SSH protocol |
| **Data Encryption** | Does not encrypt data before sending | Encrypts data before sending |
| **Transfer Method** | Direct method | Tunneling method |
| **Security** | Uploading and downloading without security | Maintains full security using SSH keys |
| **Number of Channels** | Uses two channels | Uses only one channel |
| **Authentication** | Does not require authentication | Requires username/password or SSH keys |
| **Speed** | Faster | Slower compared to FTP |
| **Data Transfer Method** | Direct transfer method | Tunneling method |
