# Essentials of Networking for DevSecOps Engineer

## Introduction to Networking in DevOps

### What is a Network?

When two or more computers and computing devices are connected together through communication channels, such as cables or wireless media, and sharing files, it is called a Network.

A network is used to:
- Allow the connected devices to communicate with each other
- Enable multiple users to share devices over the network, such as music and video servers, printers and scanners

**The Internet** is the largest network in the world and can be called "the network of networks".

### Networking Overview for DevOps

Networking is a fundamental component of IT infrastructure, and it is crucial for DevOps engineers to understand how networks operate. Networking in DevOps involves managing and optimizing the flow of data between different systems, applications, and environments.

#### Importance of Networking in DevOps

- Ensures smooth communication between microservices and applications
- Supports the scalability and performance of cloud infrastructure
- Enhances security by controlling access and data flow
- Facilitates automation and continuous integration/continuous deployment (CI/CD) processes

## Types of Networks

There are different types of networks, but the main two are LAN and WAN:

| Type | Description | Examples |
|------|-------------|----------|
| **LAN** (Local Area Network) | Interconnects computers within a limited area, such as residences, schools | Wi-Fi, Ethernet |
| **MAN** (Metropolitan Area Network) | Used in metropolitan areas (cities) | - |
| **WAN** (Wide Area Network) | Extends LAN over a large geographic area | Optical fiber cable |
| **SONET** (Synchronous Optical Network) | Used in submarine cables | - |

# OSI Model

The **Open Systems Interconnection (OSI) model** is a conceptual framework that describes how data moves through a network. OSI is a layered approach that defines seven layers, each with a specific function in the communication process.

The OSI model is crucial in DevOps networking because it provides a common reference point for network architects, engineers, and administrators to design, implement, and troubleshoot networks.

## The Seven Layers of the OSI Model

### a. Layer 1: Physical

- The first and bottom layer of the OSI model
- Converts data into a stream of bits and transmits them over the physical network medium
- Defines physical characteristics: cable type, connectors, signaling methods
- Establishes, maintains, and terminates physical connections between network devices
- Uses various communication channels: copper wires, Wi-Fi, Bluetooth, fiber optic cables
- Topologies: point-to-point, bus, star, mesh, and ring
- Protocols: Ethernet, Wi-Fi, Bluetooth, USB, HDMI

### b. Layer 2: Data Link

- Responsible for error-free transfer of data frames between devices on the same network segment
- Breaks down data from Network layer into frames and adds header/trailer
- Uses physical addresses (MAC addresses) to identify devices
- Provides framing, error detection, and flow control
- Topologies: point-to-point and broadcast
- Protocols: Ethernet, Token Ring, FDDI

### c. Layer 3: Network

- Responsible for logical addressing and routing data between different networks
- Provides end-to-end communication between two hosts on different networks
- Adds header with logical addressing information (IP addresses)
- Provides routing services using various routing algorithms
- Manages congestion control
- Protocols: Internet Protocol (IP), ICMP, IGMP
- Devices: routers and switches

### d. Layer 4: Transport

- Provides reliable end-to-end delivery of data between applications or hosts on different networks
- Breaks data segments into smaller units called packets
- Regulates flow of data and controls congestion
- Protocols: TCP, UDP, SCTP

### e. Layer 5: Session

- Establishes and manages sessions between applications on different devices
- Maintains session state, tracking conversation and data exchange
- Allows synchronized data exchange and recovery from communication failures
- May provide encryption, compression, and checkpoints
- Protocols: NetBIOS, RPC, SQL

### f. Layer 6: Presentation

- Ensures data exchanged between applications is presented in standardized format
- Performs data transformation (converting formats for compatibility)
- Performs encryption and decryption
- Performs compression and decompression
- Protocols: JPEG, GIF, MPEG, ASCII, EBCDIC

### g. Layer 7: Application

- Top layer where user applications interact with the network
- Provides user interface and services for applications to exchange information
- Supports email, web browsing, file transfer, and remote access
- Protocols: HTTP, FTP, SMTP, Telnet
- Handles application-specific functions like message formatting, file compression, user authentication

# TCP/IP

TCP/IP is a set of communication protocols that allow computers to communicate over the internet.

### What is TCP/IP, and How is it Different from OSI?

TCP/IP is the most widely used networking protocol suite in DevOps networking. It provides a reliable, connection-oriented communication model for transmitting data between devices.

**Key points about TCP/IP:**
- TCP provides reliable and ordered delivery of data
- IP provides routing and addressing capabilities
- Responsible for breaking data into packets, routing packets, and reassembling them

**The Four Layers of TCP/IP:**

| Layer | Function |
|-------|----------|
| **Network Interface Layer** | Transmits data over physical network (hardware addressing, error detection, media access control) |
| **Internet Layer** | Addresses and routes data packets among devices on distinct networks |
| **Transport Layer** | Provides end-to-end data transport (error detection, flow control, congestion control) |
| **Application Layer** | Handles protocols and processes for application communication |

### Differences Between TCP/IP and OSI Model

| Feature | TCP/IP | OSI |
|---------|--------|-----|
| Number of Layers | 4 layers | 7 layers |
| Functionality | Layers focused on specific tasks | Layers are more generalized |
| Implementation | Practical implementation | Theoretical framework |
| Usage | Suite of protocols for internet communication | Conceptual model describing communication |

### TCP vs UDP

| Feature | TCP | UDP |
|---------|-----|-----|
| Connection | Connection-oriented (establishes connection before exchanging data) | Connectionless (no dedicated connection) |
| Reliability | Reliable (error detection, flow control, retransmission) | Unreliable (no guarantees of delivery) |
| Speed | Slower | Faster |
| Acknowledgment | Yes (receiver acknowledges segments) | No |
| Flow/Congestion Control | Yes | No |
| Best For | Long-lived connections, small data bursts | Short-lived connections, large data bursts |
| Use Cases | Email, file transfer, web browsing | Video streaming, online gaming |

### Ports and Protocols

**What is a Port?**
A port is a communication endpoint for identifying specific applications and services on a device.

**Port Number Ranges:**
- **Well-known ports:** 0-1023 (associated with services/applications)
- **Registered ports:** 1024-49151
- **Dynamic/private ports:** 49152-65535 (temporary purposes)

**Common Well-Known Ports and Protocols:**

| Protocol | Port |
|----------|------|
| HTTP | 80 |
| HTTPS | 443 |
| SSH | 22 |
| FTP | 20/21 |
| SMTP | 25 |
| NTP | 123 |
| IMAP | 143 |
| SNMP | 161 |
| DNS | 53 |
| DHCP | 67/68 |
| MySQL Database | 3306 |
| Telnet (unsecured) | 23 |

**Common Protocols:**

| Protocol | Description |
|----------|-------------|
| **HTTP** | Hypertext Transfer Protocol - foundation of data communication on the web |
| **HTTPS** | HTTP with SSL/TLS encryption for secure data transmission |
| **TCP** | Reliable, connection-oriented protocol with error detection/recovery |
| **UDP** | Connectionless protocol with fast but unreliable transmission |
| **DNS** | Domain Name System - resolves domain names to IP addresses |
| **SMTP** | Protocol for sending email messages between servers |
| **SSH** | Secure Shell - secure remote server access with encrypted communication |

# Networking Tools for DevOps Engineers

| Tool | Purpose | Usage Example |
|------|---------|---------------|
| **ping** | Check reachability of a host on an IP network | `ping google.com` |
| **traceroute** | Display route and measure transit delays | `traceroute google.com` |
| **netstat** | Display active network connections, routing tables, interface statistics | `netstat -a` |
| **nmap** | Discover hosts and services on a computer network | `nmap -p 1-1000 target` |
| **tcpdump** | Capture and analyze network traffic | `tcpdump -i eth0` |
| **ifconfig** (Linux) / **ipconfig** (Windows) | Display network interface configuration | `ifconfig` |
| **dig** | Query DNS name servers for information | `dig google.com` |
| **host** (Linux) / **nslookup** (Windows) | Query DNS servers for domain information | `host google.com` |
| **Wireshark** | Network protocol analyzer for troubleshooting | Capture/analyze packets on network interface |
| **iperf** | Measure TCP and UDP network performance | `iperf -s` (server), `iperf -c` (client) |

## Network Monitoring Tools

Use tools like **Nagios**, **Zabbix**, and **Cacti** to monitor network performance, availability, and uptime. These provide visibility into network traffic and detect anomalies such as:
- Traffic spikes
- Unusual traffic patterns
- Potential security threats

# IP Subnetting and CIDR

## IP Subnetting

Subnetting means dividing a relatively large network into smaller subnetworks, or subnets. It involves creating subnets with their unique network addresses and configuring them to work together as a single network.

### Benefits of Subnetting

| Benefit | Description |
|---------|-------------|
| **Efficient IP allocation** | Reduces wastage of IP addresses |
| **Enhanced security** | Isolates sensitive resources and limits access |
| **Improved performance** | Enables efficient traffic management, reduces congestion |

### Key Subnetting Concepts

| Concept | Description | Example |
|---------|-------------|---------|
| **Subnet Mask** | Numerical value identifying network/host portions of an IP address | `255.255.255.0` (first three octets = network, last = host) |
| **Network ID** | Part of IP address identifying the network | For `192.168.1.10` with mask `255.255.255.0`, network ID = `192.168.1.0` |
| **Host ID** | Part of IP address specifying a particular device | For same address, host ID = `0.0.0.10` |
| **CIDR Notation** | Shorthand method of representing subnet mask | `192.168.1.0/24` |

## CIDR (Classless Inter-Domain Routing)

CIDR is a method used to allocate and manage IP addresses more efficiently.

**Key features of CIDR:**
- Enables variable-length subnet masks to create subnets of different sizes
- Denoted by a slash followed by a number (e.g., `/24`)
- Helps conserve IP addresses by allowing smaller subnets
- Enables easier route summarization, reducing routing table size

### Common CIDR Notations

| CIDR Notation | Subnet Mask | Total Hosts | Usable Hosts | Networks |
|---------------|-------------|-------------|--------------|----------|
| `/24` | 255.255.255.0 | 256 | 254 | 1 |
| `/20` | 255.255.240.0 | 4,096 | 4,094 | 16 |
| `/16` | 255.255.0.0 | 65,536 | 65,534 | 256 |
| `/8` | 255.0.0.0 | 16,777,216 | 16,777,214 | 256 |

# Routing

Routing is a crucial aspect of networking that sends data packets from one network to another.

### Key Routing Concepts

- **Routers** perform routing in a network
- Routers use **routing tables** to determine the best path for a packet
- **Next-hop address** is the IP address of the next router on the path

### Types of Routing

| Type | Description |
|------|-------------|
| **Static Routing** | Routes are manually configured and don't change unless explicitly modified |
| **Dynamic Routing** | Routing protocols automatically determine routes based on network topology and traffic conditions |

### Routing Protocols

| Category | Protocols |
|----------|-----------|
| **Interior Routing** | OSPF, EIGRP, RIP |
| **Exterior Routing** | BGP (Border Gateway Protocol) |

**Additional routing features:**
- **Load balancing** - Distributes network traffic across multiple paths to avoid congestion
- **BGP** - Used by ISPs to exchange routing information between networks

# Domain Name System (DNS)

DNS is a hierarchical and decentralized naming system for computers, services, or resources connected to the internet or a private network. It works like a phonebook, converting human-readable domain names (e.g., google.com) into IP addresses (e.g., 172.217.6.14).

### DNS Overview

- Distributed database containing information about domain names and associated IP addresses
- Operates at the Application layer of the OSI model
- Uses client-server model

### DNS Resolution Process

1. User types domain name into browser
2. Browser requests DNS resolver to find corresponding IP address
3. Resolver checks its cache
4. If not in cache, resolver sends request to root server
5. Root server responds with TLD server address
6. Resolver queries TLD server for IP address

### Domains, Zones, and Delegation

| Concept | Description |
|---------|-------------|
| **Domain** | Hierarchical way of organizing resources; e.g., example.com with subdomains like blog.example.com |
| **Zone** | Contiguous portion of DNS namespace managed by a single entity; contains domain info (name servers, mail servers, etc.) |
| **Delegation** | Transferring responsibility for managing a subdomain to another DNS server |

### DNS Record Types

| Record Type | Description |
|-------------|-------------|
| **A** | Maps domain name to IPv4 address |
| **AAAA** | Maps domain name to IPv6 address |
| **CNAME** | Maps alias name to canonical name (another domain name) |
| **MX** | Maps domain name to mail exchange server |
| **NS** | Specifies authoritative DNS server for the domain |
| **SOA** | Specifies authoritative info about a DNS zone |
| **TXT** | Stores text-based information associated with a domain name |
| **PTR** | Maps IP address to domain name (reverse lookup) |

# Hypertext Transfer Protocol (HTTP)

HTTP is the foundation of data communication on the World Wide Web. It is a request-response protocol essential for building, maintaining, and troubleshooting web applications and services.

## HTTP Methods

### HTTP Request Methods

| Method | Description |
|--------|-------------|
| **GET** | Retrieves a resource from the server |
| **POST** | Submits data to be processed to a specific resource |
| **PUT** | Updates a resource on the server |
| **DELETE** | Removes a resource from the server |
| **PATCH** | Partially updates a resource on the server |

## HTTP Response Codes

### Categories of HTTP Responses

| Category | Description | Examples |
|----------|-------------|----------|
| **1xx: Informational** | Server has received request and is processing | 100 Continue, 101 Switching Protocols |
| **2xx: Success** | Request successfully received, understood, and accepted | 200 OK, 201 Created, 204 No Content |
| **3xx: Redirection** | Client needs to take further action | 301 Moved Permanently, 302 Found, 307 Temporary Redirect |
| **4xx: Client Error** | Client-side errors | 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found |
| **5xx: Server Error** | Server-side errors | 500 Internal Server Error, 503 Service Unavailable |

### Common HTTP Response Codes

| Code | What It Means | Description |
|------|---------------|-------------|
| 200 | OK | Request succeeded |
| 301, 308 | Moved Permanently, Permanent Redirect | URI of requested resource changed permanently |
| 302, 307 | Found, Temporary Redirect | URI changed temporarily |
| 400 | Bad Request | Server can't understand the request |
| 401 | Unauthorized | Client must authenticate |
| 403 | Forbidden | Client lacks permission |
| 404 | Not Found | Server can't find requested resource |
| 408 | Request Timeout | Response sent to idle connection |
| 418 | I am a Teapot | Server refuses to brew coffee (it's a teapot!) |
| 500 | Internal Server Error | Server doesn't know how to handle request |
| 502 | Bad Gateway | Gateway received incorrect response from source server |
| 503 | Service Unavailable | Server is down or overloaded |
| 504 | Gateway Timeout | Gateway can't get response in time |

## HTTP Headers

HTTP headers provide important information about the client, server, and content being transmitted.

### Types of HTTP Headers

| Header Type | Description | Examples |
|-------------|-------------|----------|
| **General Headers** | Apply to both requests and responses | Request URL, Request Method, Status Code |
| **Request Headers** | Provide information about the client or requested resource | User-Agent, Accept, Authorization |
| **Response Headers** | Provide information about the server response | Content-Type, Content-Length, Server |
| **Entity Headers** | Describe the content of the message body | Content-Encoding, Content-Language, Content-Disposition |

# Virtual Private Network (VPN)

A **Virtual Private Network (VPN)** establishes a secure connection over an untrusted network (the internet). By creating a logical tunnel through which data is encrypted, VPNs enable users to send and receive data securely, regardless of their location.

## Types of VPN

| Type | Description |
|------|-------------|
| **Site-to-Site VPN** | Connects entire networks to each other, allowing multiple sites/offices to communicate securely |
| **Remote Access VPN** | Allows individuals to connect to their organization's network securely (ideal for remote work) |
| **SSL VPN** | Uses Secure Sockets Layer for remote access with standard web protocols |
| **IPsec VPN** | Encrypts data at the IP layer, ensuring secure device communication |

## Key Features of VPN

| Feature | Description |
|---------|-------------|
| **Confidentiality** | Only authorized parties can access transmitted data (achieved through encryption) |
| **Integrity** | Data remains unchanged during transmission (using hashing algorithms) |
| **Authentication** | Validates identities of devices or users connected to the VPN |
| **Replay Protection** | Prevents attackers from capturing and re-sending packets (using sequence numbers and timestamps) |

## How a VPN Works

A VPN operates by establishing a secure, encrypted tunnel between your device and a remote server controlled by the VPN service.

### Encryption and Tunneling

- **Encryption**: Data is encrypted before leaving your device (transformed into ciphertext)
- **Common encryption standards**: AES with 128-bit or 256-bit keys
- **Tunneling**: Encapsulates data within another set of data packets

## VPN Protocols

| Protocol | Description |
|----------|-------------|
| **OpenVPN** | Open-source protocol balancing security and speed; uses SSL/TLS for key exchange |
| **L2TP/IPsec** | L2TP combined with IPsec for encryption; slower due to double encapsulation |
| **WireGuard** | Newer protocol known for simplicity, speed, and modern cryptography |
| **IKEv2/IPsec** | Effective for mobile users; stable when switching between networks |

## Practical Applications of VPN

- **Remote Work**: Secure access to company resources and applications
- **Secure Communication**: Protecting sensitive information from data breaches
- **Bypassing Geographical Restrictions**: Masking IP addresses to access region-blocked content

## Configuration Considerations

- Choosing the right protocol (IPsec, SSL, etc.)
- Defining policies (encryption algorithms, authentication methods, integrity checks)
- Ongoing management (regular monitoring and updates)
