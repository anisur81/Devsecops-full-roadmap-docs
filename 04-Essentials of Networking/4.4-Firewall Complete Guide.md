# Firewall Complete Guide

## Introduction to Firewall

Firewall details typically refer to the configuration and status information of a firewall that controls incoming and outgoing network traffic. Here are the key components you'll usually see:

### 🔒 Basic Firewall Details

- **Status:** Enabled or disabled
- **Type:** Hardware firewall (router-based) or software firewall (OS/application-based)
- **Default Policy:** Allow or deny by default

### 📡 Rules / Policies

- **Inbound Rules:** Control traffic coming into your system
- **Outbound Rules:** Control traffic going out
- **Allowed/Blocked Ports:** e.g., 80 (HTTP), 443 (HTTPS), 22 (SSH)
- **IP Filtering:** Allow or block specific IP addresses or ranges
- **Protocol Filtering:** TCP, UDP, ICMP, etc.

### 🧱 Security Features

- **Stateful Inspection:** Tracks active connections
- **Packet Filtering:** Basic filtering based on headers
- **Application Layer Filtering:** Controls traffic by application (e.g., browsers)
- **Intrusion Detection/Prevention (IDS/IPS)**

### 📊 Logs & Monitoring

- Traffic logs (allowed/blocked attempts)
- Alerts for suspicious activity
- Connection history

### 🌐 Network Zones

- **LAN** (internal network)
- **WAN** (external internet)
- **DMZ** (demilitarized zone) for public-facing services

### ⚙️ Advanced Settings

- **NAT** (Network Address Translation)
- **Port forwarding**
- **VPN support**
- **QoS** (traffic prioritization)

## UFW (Uncomplicated Firewall)

**UFW (Uncomplicated Firewall)** is a user-friendly command-line tool used to manage firewall rules on Linux systems, especially on Ubuntu-based distributions.

A firewall controls incoming and outgoing network traffic to improve system security.

### Simple Definition

UFW helps administrators easily:

- Allow network connections
- Block unauthorized access
- Open or close ports
- Secure servers and applications

---

## Install UFW Firewall

UFW comes pre-installed with Ubuntu operating system. However, you can make sure that you have the latest version of UFW by trying to install it using the following command:

```bash
apt install ufw
```

Now that you have the latest version of UFW installed, let's check its status:

```bash
ufw status
```

To enable UFW on your system, run:

```bash
sudo ufw enable
```

---

## UFW Default Policies

When you enable UFW for the first time, it applies a set of default policies that define how the firewall handles incoming and outgoing connections. By default, UFW is configured to:

- **Deny** all incoming connections
- **Allow** all outgoing connections

You can verify the current default policy settings with the following command:

```bash
sudo ufw status verbose
```

**Sample output:**
```
Status: active
Logging: on (low)
Default: deny (incoming), allow (outgoing), deny (routed)
New profiles: skip
```

### Changing Default Policies

If you want to change the default behavior, you can update the default policies using the following commands:

To deny all incoming connections (recommended for most servers):

```bash
sudo ufw default deny incoming
```

To allow all outgoing connections (the default):

```bash
sudo ufw default allow outgoing
```

To deny routed connections:

```bash
sudo ufw default deny routed
```

---

## How to Disable UFW

If, for some reason, you need to disable UFW, you can do so with the following command:

```bash
sudo ufw disable
```

To check whether UFW is managing your iptables rules, you can inspect the raw iptables output:

```bash
sudo iptables -L
```

UFW stores its persistent rule definitions in configuration files, typically under:

```
/etc/ufw/
```

## SSH Configuration

If you have configured SSH to use another port, use a more specific command to create an allow rule for SSH connections. For instance, if SSH service is listening to port 4422:

```bash
ufw allow 4422/tcp
```

This firewall rule allows TCP connections to port 4422.

---

## How to Block an IP Address

To block all network connections that originate from a specific IP address:

```bash
sudo ufw deny from 203.0.113.100
```

---

## How to Block a Subnet

If you need to block a full subnet, use the subnet address with the `from` parameter:

```bash
sudo ufw deny from 203.0.113.0/24
```

This would block all IP addresses in the example subnet.

---

## How to Block Incoming Connections to a Network Interface

To block incoming connections from a specific IP address to a specific network interface:

```bash
sudo ufw deny in on eth0 from 203.0.113.100
```

---

## How to Allow an IP Address

To allow all network connections that originate from a specific IP address:

```bash
sudo ufw allow from 203.0.113.101
```

---

## How to Allow Incoming Connections to a Network Interface

To allow incoming connections from a specific IP address to a specific network interface:

```bash
sudo ufw allow in on eth0 from 203.0.113.102
```

---

## How to Delete UFW Rules

### Method 1: Delete by rule specification

To delete a rule that you previously set up, use `ufw delete` followed by the rule:

```bash
sudo ufw delete allow from 203.0.113.101
```

### Method 2: Delete by rule ID

First, get the rule numbers:

```bash
sudo ufw status numbered
```

Then delete by ID:

```bash
sudo ufw delete 1
```

## Add UFW Firewall Rules

There are various firewall rules that you can apply to your system through UFW:

| Rule Type | Description |
|-----------|-------------|
| `allow` | Allow traffic |
| `deny` | Silently discard traffic |
| `reject` | Reject traffic and send back an error packet to the sender |
| `limit` | Limit connections from a specific IP address that has attempted to initiate 6 or more connections in the last 30 seconds |

## How to Allow Incoming SSH from a Specific IP Address or Subnet

The following command will allow only SSH connections coming from a specific IP address:

```bash
sudo ufw allow from 203.0.113.103 proto tcp to any port 22
```

To allow incoming SSH connections from an entire network:

```bash
sudo ufw allow from 203.0.113.0/24 proto tcp to any port 22
```

## How to Allow a Port Range

If your application uses a range of ports for different activities, whitelist a port range:

```bash
ufw allow 8080:9090/tcp
```

## How to Reset UFW to Default Configuration

Resetting UFW will:

- Disable the firewall (UFW will no longer be active)
- Delete all existing rules (both allow and deny)
- Reset default policies to:
  - `deny` for all incoming traffic
  - `allow` for all outgoing traffic

```bash
sudo ufw reset
```

---

## How to Back Up Rules Before Making Major Changes

Before resetting or reworking your UFW configuration, save the current ruleset:

```bash
sudo ufw status numbered > ufw-backup.txt
```

## Use Logging to Monitor Activity

### Enable Logging

Enable logging to capture dropped connections and debug firewall behavior:

```bash
sudo ufw logging on
```

Logs are typically stored in:

```
/var/log/ufw.log
```

### Set UFW Logging Level

There are five UFW logging levels. Each level collects increasingly more data.

> **Note:** Log levels above `medium` may generate a lot of logging output and may quickly fill up a disk on a busy system. Use them carefully.

| Level | Description |
|-------|-------------|
| `off` | Disables UFW managed logging |
| `low` | Logs all blocked packets not matching default policy, plus packets matching pre-set rules |
| `medium` | Same as low, plus all allowed packets not matching defined policy, all invalid packets, and all new connections |
| `high` | Same as medium, just without rate limiting, plus all packets with rate limiting |
| `full` | Same as high, just without rate limiting |

### Control verbosity:

```bash
sudo ufw logging low   # or medium / high / full
```
