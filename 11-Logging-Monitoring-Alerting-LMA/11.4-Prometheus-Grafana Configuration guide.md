## Part 2: Prometheus Configuration in Istio

### How Prometheus Works in Istio Mesh

In an Istio mesh, each component exposes an endpoint that emits metrics. Prometheus works by scraping these endpoints and collecting the results. This is configured through the Prometheus configuration file which controls:
- Which endpoints to query
- Port and path to query
- TLS settings
- Scrape intervals and timeouts

### Components Scraped by Prometheus

To gather metrics for the entire mesh, Prometheus scrapes:

| Component | Description |
|-----------|-------------|
| Control Plane | `istiod` deployment |
| Gateways | Ingress and Egress gateways |
| Envoy Sidecar | Proxy metrics |
| User Applications | If they expose Prometheus metrics |

## Part 3: Metrics Merging Option

### Option 1: Metrics Merging (Simplified Configuration)

Istio has the ability to control scraping entirely by `prometheus.io` annotations, allowing Istio scraping to work out of the box with standard configurations.

**Check if enabled:**
```bash
kubectl -n istio-system get configmap istio -o jsonpath='{.data.mesh}' | grep enablePrometheusMerge
```

**Expected output:**
```
enablePrometheusMerge: true
```

> **Note:** This option is enabled by default but can be disabled by passing `--set meshConfig.enablePrometheusMerge=false` during installation.

## Part 5: Verifying Prometheus Scraping

### Check Targets in Prometheus UI

1. **Open Prometheus UI** (port-forward if needed):
```bash
kubectl port-forward svc/prometheus -n istio-system 9090:9090
```

2. Navigate to **Status → Targets**

**You should see healthy targets:**
- ✅ istiod
- ✅ envoy-stats
- ✅ kubernetes-pods
- ✅ kubernetes-service-endpoints

**Status should show:** `UP`

### Test Metrics Immediately

In Prometheus query bar, run:
```promql
istio_requests_total
```

**If working, you will see:**
- Increasing values over time
- Labels like `source_service`, `destination_service`, `response_code`

## Part 6: Useful PromQL Queries

### Request Rate by Service
```promql
sum(rate(istio_requests_total{reporter="destination"}[5m])) by (destination_service)
```

### Error Rate by Service (5xx errors)
```promql
sum(rate(istio_requests_total{reporter="destination", response_code=~"5.."}[5m]))
by (destination_service)
/
sum(rate(istio_requests_total{reporter="destination"}[5m]))
by (destination_service)
```

### P99 Latency by Service
```promql
histogram_quantile(0.99,
  sum(rate(istio_request_duration_milliseconds_bucket{reporter="destination"}[5m]))
  by (le, destination_service)
)
```

### Request Rate Between Specific Services
```promql
sum(rate(istio_requests_total{
  reporter="destination",
  source_workload="productpage-v1",
  destination_workload="reviews-v1"
}[5m]))
```

### TCP Connection Rate
```promql
sum(rate(istio_tcp_connections_opened_total{reporter="destination"}[5m]))
by (destination_service)
```

### Successful Request Rate (200 status)
```promql
sum(rate(istio_requests_total{reporter="destination", response_code="200"}[5m]))
```

## Part 9: Configure Prometheus Data Source in Grafana

### Step-by-Step Configuration

1. **Go to Settings** (⚙️) → **Data Sources**
2. Click **Add data source**
3. Select **Prometheus**
4. Set URL (choose one based on your installation):
   - `http://prometheus-server.monitoring.svc.cluster.local:80`
   - `http://prometheus-k8s.monitoring.svc:9090`
   - `http://prometheus.istio-system.svc:9090`
5. Click **Save & Test**

**Expected result:** `Data source is working`

## Part 13: Troubleshooting

### No Metrics in Prometheus

**Check if metrics merging is enabled:**
```bash
kubectl -n istio-system get configmap istio -o jsonpath='{.data.mesh}' | grep enablePrometheusMerge
```

**Verify Envoy sidecar is injecting:**
```bash
kubectl get pod -n <namespace> <pod-name> -o jsonpath='{.spec.containers[*].name}'
```

**Check Envoy metrics endpoint:**
```bash
kubectl exec -n <namespace> <pod-name> -c istio-proxy -- curl localhost:15000/stats/prometheus | head
```

### Prometheus Cannot Scrape

**Check network policies:**
```bash
kubectl get networkpolicies -A
```

**Verify service endpoints:**
```bash
kubectl get endpoints -n istio-system
```

### Grafana Cannot Connect to Prometheus

**Test connectivity from Grafana pod:**
```bash
kubectl exec -n istio-system <grafana-pod> -- wget -O- http://prometheus.istio-system.svc:9090/-/healthy
```

## Part 14: Best Practices

### Production Recommendations

1. **Use Persistent Storage** for Prometheus metrics
2. **Implement Retention Policies** to manage storage
3. **Set Resource Limits** for both Prometheus and Grafana
4. **Enable Authentication** for Grafana
5. **Use HTTPS/Ingress** instead of port-forwarding
6. **Configure Alertmanager** for production alerts
7. **Implement High Availability** for critical monitoring

### Performance Optimization

```yaml
# Prometheus resource limits
resources:
  requests:
    memory: 2Gi
    cpu: 1
  limits:
    memory: 4Gi
    cpu: 2

# For large clusters, increase scrape interval
scrape_interval: 60s

# Use recording rules for expensive queries
```

### Security Hardening

```yaml
# Grafana with authentication
auth:
  disable_login_form: false
  disable_signout_menu: false

# Prometheus with basic auth or OAuth
basic_auth:
  username: prometheus
  password: <secure-password>
```
