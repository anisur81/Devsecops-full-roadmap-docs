 # OpenTelemetry with Istio: Complete Implementation Guide

## What is OpenTelemetry?

**OpenTelemetry (OTel)** is a vendor-neutral, open source observability framework for instrumenting, generating, collecting, and exporting telemetry data.

**OpenTelemetry Protocol (OTLP)** traces can be sent to:
- Jaeger
- Prometheus
- Grafana Tempo
- Commercial services (Datadog, New Relic, Dynatrace, etc.)

## Part 1: OpenTelemetry Collector Configuration

### Verify OTel Collector is Exporting to Jaeger

Check the OTel configuration file:

```bash
cat istio-1.30.0/samples/open-telemetry/otel.yaml
```

### Sample OTel Configuration

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: opentelemetry-collector-conf
  labels:
    app: opentelemetry-collector
data:
  opentelemetry-collector-config: |
    receivers:
      opencensus:
        endpoint: 0.0.0.0:55678
      otlp:
        protocols:
          grpc:
            endpoint: 0.0.0.0:4317
          http:
            endpoint: 0.0.0.0:4318

    processors:
      batch: {}

    exporters:
      otlp:
        endpoint: jaeger-collector.istio-system.svc.cluster.local:4317
        tls:
          insecure: true
      debug:
        verbosity: detailed

    extensions:
      health_check: {}
      pprof: {}
      zpages: {}

    service:
      extensions:
        - health_check
        - pprof
        - zpages
      pipelines:
        traces:
          receivers:
            - otlp
          processors:
            - batch
          exporters:
            - otlp
            - debug
        logs:
          receivers:
            - otlp
          processors:
            - batch
          exporters:
            - debug
```

## Part 3: Configure Istio for OTel Tracing

### Enable Tracing via IstioOperator

```bash
cat <<EOF | istioctl install -y -f -
apiVersion: install.istio.io/v1alpha1
kind: IstioOperator
spec:
  meshConfig:
    enableTracing: true
    extensionProviders:
    - name: otel-tracing
      opentelemetry:
        port: 4317
        service: opentelemetry-collector.observability.svc.cluster.local
        resource_detectors:
          environment: {}
EOF
```

## Part 9: Understanding Your Tracing Architecture

### Why Port 4317 is NOT Present in Proxy Config

**Reasoning:**

1. **Not using direct OTLP gRPC from Envoy** - Istio forwards tracing through a gateway abstraction
2. **Istio abstraction layer** - Envoy only sees a service name, not the port
3. **OTLP collector hidden** - Behind the gateway for better abstraction

### Why Tracing Still Works

Tracing works because Istio handles it at a higher abstraction level:

- Istio manages tracing injection via:
  - Telemetry resource
  - extensionProviders
  - Envoy tracing driver config

**Envoy only needs:**
```
bookinfo-otel-gateway-istio.gateway-system.svc.cluster.local
```

### What "EDS" Means

**Endpoint Discovery Service** - Dynamic service resolution

**Implications:**
- Envoy does NOT hardcode IP or port
- It resolves the gateway dynamically via Istio control plane
- Enables automatic service discovery and load balancing

## Part 10: Troubleshooting

### No Traces in Jaeger

**Verify OTel Collector is receiving traces:**
```bash
kubectl logs -n observability deployment/opentelemetry-collector --tail=50
```

**Check Telemetry configuration:**
```bash
kubectl describe telemetry otel-demo -n istio-system
```

**Verify Istio mesh config:**
```bash
kubectl get configmap istio -n istio-system -o yaml | grep -A20 "mesh:"
```

### OTel Collector Cannot Connect to Jaeger

**Test connectivity:**
```bash
kubectl exec -n observability deployment/opentelemetry-collector -- \
  wget -O- http://jaeger-collector.istio-system.svc.cluster.local:4317
```

### Pods Not Injected with Sidecar

**Verify namespace labeling:**
```bash
kubectl get namespace observability -o yaml | grep istio-injection
```

**Manually restart deployments:**
```bash
kubectl rollout restart deployment -n observability
```

## Part 11: Best Practices

### Sampling Strategy

For production, adjust sampling percentage:
```yaml
randomSamplingPercentage: 10  # 10% sampling in production
```

### Resource Detection

Leverage environment resource detectors:
```yaml
resource_detectors:
  environment: {}
  kubernetes: {}  # Add Kubernetes metadata
```

### Batch Processing

Configure batch processor for performance:
```yaml
processors:
  batch:
    timeout: 1s
    send_batch_size: 1024
```

### Exporters Configuration

Use multiple exporters for redundancy:
```yaml
exporters:
  otlp/jaeger:
    endpoint: jaeger-collector.istio-system.svc.cluster.local:4317
  otlp/tempo:
    endpoint: tempo.observability.svc.cluster.local:4317
  debug:  # For development only
    verbosity: basic
```
